# 要求定義: サイドバー内の目次レイアウト改善 (toc-layout)

## 背景・動機

目次はサイドバー下部に**固定高さ（デフォルト200px）**で表示されるため、ウィンドウが縦に大きいほど目次の占める比率が小さくなり、「表示位置が下過ぎる」印象になる（2026-07-09 ユーザーフィードバック）。エクスプローラーと目次の高さ配分を、絶対値ではなく比率で管理する。

## スコープ

- **対象**: サイドバー内のエクスプローラー／目次の高さ配分、目次が空のときの表示
- **対象外**: 目次の配置変更（右パネルへ戻す等）、目次自体の内容・階層表示の変更

---

## ユーザーストーリー

### US-001: 比率ベースの高さ配分

```
As a ユーザー
I want to エクスプローラーと目次の高さがウィンドウサイズに応じた比率で配分される
So that 大きなウィンドウでも目次が十分な高さで表示される
```

**受け入れ条件**

- [x] デフォルトでエクスプローラー約30%：目次約70%の比率で表示される
- [x] ウィンドウ（サイドバー）の高さを変えても比率が維持される
- [x] リサイズハンドルのドラッグで比率を変更でき、変更した比率は永続化される
- [x] リサイズハンドルのダブルクリックでデフォルト比率に戻る

### US-002: 目次が空のときは自動で畳む

```
As a ユーザー
I want to 目次に表示する内容がないときはエクスプローラーが全高になる
So that フォルダ閲覧時に空の目次領域で画面が無駄にならない
```

**受け入れ条件**

- [x] ファイルを開いていない、または開いているファイルに見出しがないとき、目次はヘッダー行のみに畳まれエクスプローラーが残り全高を使う
- [x] 見出しのあるファイルを開くと設定された比率で目次が表示される
- [x] 目次表示トグル（設定・ショートカット）の挙動は従来どおり
