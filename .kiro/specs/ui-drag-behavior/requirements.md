# 要求定義: UIドラッグ挙動の改善 (ui-drag-behavior)

## 背景・目的

Sidebar・目次のリサイズ後に、ポインター操作で得たフォーカスリングが太線として残る。また、タブやExplorer、目次などアプリ操作用UIの文字列をドラッグで選択でき、デスクトップアプリとして不要な選択表示が発生する。

本文を読む・コピーする操作と入力欄の編集性は維持しつつ、アプリUIのドラッグ挙動を自然にする。

## 要求

### R-001 リサイズ終了後の表示

- [x] Sidebar・目次で共用するリサイズハンドルは、ポインターを離した後に太いフォーカスリングを残さない。
- [x] `pointerup`、`pointercancel`、pointer capture喪失のいずれでもドラッグ状態を終了し、最終幅をコミットする。
- [x] キーボードでTabフォーカスした場合は従来どおりフォーカスリングを表示する。
- [x] 主ポインターの左ボタン以外ではドラッグを開始しない。

### R-002 アプリUIの文字選択抑止

- [x] タブ、Explorer、目次、ステータスバー、設定、ウェルカム・読み込み表示等のアプリUIはドラッグで文字選択されない。
- [x] 実文書を表示するレンダリング済みMarkdown、Markdownソース、セーフモード本文だけは従来どおり選択・コピーできる。
- [x] `input`、`textarea`、編集可能要素内の文字は従来どおり選択できる。

## 対象外

- Markdown本文の選択・コピー操作の変更
- OSウィンドウ枠やネイティブメニューのドラッグ挙動
- タブの並べ替えやライトボックスのパン操作の仕様変更
