# 要求定義: クイックオープンとコマンドパレット

## 背景・目的

Explorerを辿らずに現在の文書ソース内のMarkdownへ移動できる経路と、機能名を覚えていれば既存コマンドを実行できる経路を提供する。両機能は検索、候補選択、キーボード操作が共通するため、同じPicker UIを利用する。

## スコープ

- ネイティブFSおよびZIPの現在のExplorerを対象とするクイックオープン
- 登録済みユーザー向けコマンドの検索・実行
- 共通Picker UI、部分一致を優先するfuzzy match、最大100件表示
- `Ctrl+P` / `Ctrl+Shift+P`およびネイティブメニューからの起動

## 対象外

- Explorerを開いていない状態でPC全体や最近使ったファイルを検索すること
- 索引の永続化、バックグラウンド常時索引、ファイル内容の検索
- コマンドへの引数入力、任意スクリプト実行、ユーザー定義コマンド
- Picker内でのファイルプレビュー

## US-001: Markdownを素早く開く

As a ユーザー
I want to ファイル名の一部から現在のExplorer内のMarkdownを開く
So that 深いフォルダーやZIP内の文書へ素早く移動できる

### 受け入れ条件

- [x] `Ctrl+P`またはメニューからクイックオープンを表示できる
- [x] ネイティブFSとZIPで、未展開のディレクトリを含む全Markdownが候補になる
- [x] 隠しファイルと`.gitignore`の扱いはExplorer設定に従う
- [x] ファイル名を主表示、ソース内相対パスを副表示する
- [x] 完全一致、前方一致、部分一致をfuzzy matchより優先する
- [x] 候補表示は上位100件までとし、上下キー・Enter・Escで操作できる
- [x] 選択した文書は既存のサイズ確認・AllowedRoots・ZIP検証を通る通常のタブとして開く
- [x] Explorer未オープン、候補なし、読込中、列挙失敗を区別して表示する

## US-002: コマンドを検索して実行する

As a ユーザー
I want to 機能名からアプリのコマンドを検索・実行する
So that メニュー位置やショートカットを覚えていなくても操作できる

### 受け入れ条件

- [x] `Ctrl+Shift+P`またはメニューからコマンドパレットを表示できる
- [x] ユーザー向けに登録されたコマンドだけを現在言語の名称で表示する
- [x] コマンド名とコマンドIDを検索対象にする
- [x] ショートカットがあるコマンドは候補に表示する
- [x] EnterまたはクリックでPickerを閉じてから既存コマンドを実行する
- [x] 内部専用コマンドやタブ番号ジャンプを一覧へ表示しない

## US-003: 共通Pickerとして一貫して操作する

As a キーボード利用者
I want to 2つのPickerを同じ操作で利用する
So that 一度覚えた操作を再利用できる

### 受け入れ条件

- [x] 開いた直後に検索入力へフォーカスする
- [x] 入力中もフォーカスをダイアログ内に保ち、閉じたら元の要素へ戻す
- [x] 選択候補をスクロール領域内へ追従させる
- [x] 背景クリックとEscで安全に閉じ、候補を実行しない
- [x] 検索語と候補文字列はテキストとして描画し、HTMLとして解釈しない

## 非機能要求

- ネイティブ走査とZIP候補数は既存の10,000エントリ上限内に収める
- ネイティブ走査はWebViewのメインスレッドをブロックしない
- 1回のPicker表示中に古い非同期結果が新しいソースの候補を上書きしない
- 新しい依存関係、永続化形式、ファイル権限を追加しない
- `Ctrl+P`はクイックオープンへ再割当し、印刷はファイルメニューとコマンドパレットから維持する
