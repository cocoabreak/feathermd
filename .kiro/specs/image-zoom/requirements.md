# 要求定義: 画像・Mermaidズーム (image-zoom)

## 背景・動機

コンテンツズーム（文字サイズ全体の拡大縮小）とは別に、画像やMermaid図など個別の要素だけを詳しく見たい場面がある。クリックした要素をライトボックス（全画面オーバーレイ）で拡大表示し、その中でさらにズーム・パンできるようにする。

## スコープ

- **対象**: Markdown本文中の `<img>` 要素、Mermaidでレンダリングされた図（SVG）のクリックによるライトボックス表示、ライトボックス内でのズーム（ホイール）・パン（ドラッグ）
- **対象外**: KaTeX数式のズーム、ライトボックスを開いたまま複数画像を切り替える機能（ギャラリー的な前後移動）、画像の回転・反転等の編集機能

---

## ユーザーストーリー

### US-001: 画像・Mermaid図のライトボックス表示

```
As a ユーザー
I want to 画像やMermaid図を拡大表示する
So that 細部まで確認できる
```

> **改訂（2026-07-03）**: 当初は画像/図本体のクリックで起動していたが、バッジ画像等での誤クリックが煩わしいとのフィードバックにより、ホバー時に表示される展開アイコンでの起動に変更する。

**受け入れ条件**

- [x] 本文中の画像・Mermaid図にマウスを乗せると右上に展開アイコンが表示される
- [x] 展開アイコンをクリックすると、全画面オーバーレイでその画像/図が拡大表示される
- [x] 画像/図本体（アイコン以外の部分）のクリックは通常のブラウザ動作に委ねる（リンクで囲まれていればリンク遷移、囲まれていなければ何も起きない）
- [x] ライトボックス表示時、初期状態では画像/図が画面に収まるサイズで表示される

---

### US-002: ライトボックス内でのズーム・パン

```
As a ユーザー
I want to ライトボックス内でさらに拡大・移動する
So that 画像/図の細部までピクセル単位で確認できる
```

**受け入れ条件**

- [x] マウスホイールでズームイン・ズームアウトできる
- [x] ドラッグで表示位置を移動できる
- [x] ダブルクリックで初期表示（画面に収まるサイズ）にリセットされる

---

### US-003: ライトボックスを閉じる

```
As a ユーザー
I want to ライトボックスを閉じて元の文書に戻る
So that 閲覧を継続できる
```

**受け入れ条件**

- [x] `Escape` キーで閉じる
- [x] 背景（画像/図以外の暗い領域）のクリックで閉じる
- [x] 閉じるボタン（✕）で閉じる

---

## 非機能要求

| 項目 | 要求 |
|------|------|
| 表示崩れ | 極端に大きい/小さい画像でもライトボックスのレイアウトが崩れない |
| 可読性 | Mermaid図（明るい背景を前提とした文字色で描画される）がライトボックス内でも文字を判読できる背景色になっている |
| 安全性 | Mermaid SVGはMarkdownレンダリング時に既にサニタイズ済みのDOMから複製するため、ライトボックス表示のために新たな未検証HTMLを挿入しない |
