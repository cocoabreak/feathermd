# 要求定義: コードブロックコピー (code-copy)

## 背景・動機

GitHubではコードブロックにマウスを乗せるとコピーアイコンが表示され、クリック一つでコード全体をクリップボードにコピーできる。このビューワーには同等の機能がなく、コードを読んで試したい場合に手動選択が必要で不便。

なお、この機能はMermaidダイアグラム専用ではなくGitHub全体の一般機能であり、通常のコードブロックにも標準で付いている。本specではまず通常のコードブロック（Mermaid以外）への対応を扱う。Mermaidダイアグラムのソースコピー・展開アイコン等のツールバーは別spec（image-zoom拡張）で扱う。

## スコープ

- **対象**: `.markdown-body` 内のフェンスコードブロック（shikiでレンダリングされる `<pre class="shiki">`）
- **対象外**: インラインコード（`` `code` ``）のコピー、Mermaidダイアグラムのコピー（別spec）、コピー成功のトースト通知等の凝ったフィードバックUI

---

## ユーザーストーリー

### US-001: コードブロックのコピー

```
As a ユーザー
I want to コードブロックのコードをワンクリックでコピーする
So that 手動選択の手間なくコードを再利用できる
```

**受け入れ条件**

- [x] コードブロックにマウスを乗せると右上にコピーアイコンボタンが表示される
- [x] クリックすると、そのコードブロックの生テキスト（シンタックスハイライトの装飾を除いたプレーンテキスト）がクリップボードにコピーされる
- [x] コードブロックを横スクロールしても、ボタンは右上に固定表示され続ける
- [x] コピー成功時、アイコンが一時的にチェックマークに変わり、数秒後に元のコピーアイコンに戻る

---

### US-002: コピー失敗時の扱い

```
As a ユーザー
I want to クリップボードへの書き込みに失敗した場合でもアプリが壊れない
So that 安定して使い続けられる
```

**受け入れ条件**

- [x] クリップボードAPI呼び出しが失敗した場合、コンソールに警告を出力するのみとし、UIはクラッシュしない

---

## 非機能要求

| 項目 | 要求 |
|------|------|
| パフォーマンス | 大量のコードブロックを含むファイルでも、DOM後処理（mermaid遅延レンダリングと同様の仕組み）でパフォーマンスに大きな影響を与えない |

---

## 未決定事項（設計フェーズで決定）

- [x] チェックマーク表示から元に戻るまでの秒数
