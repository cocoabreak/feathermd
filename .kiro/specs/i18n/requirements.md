# 要求定義: 多言語対応 (i18n)

## 背景・動機

UI文言が日本語ハードコードのため、日本語話者以外が使えない。
公開を見据え、日本語・英語の2言語に対応する。

2026-07-10に設計論点を確定して着手。

## スコープ

- **対象言語**: 日本語（ja）・英語（en）
- **対象範囲**:
  - フロントエンドの全UI文言（ラベル・ツールチップ・プレースホルダー・ダイアログ・コンテキストメニュー）
  - ネイティブメニュー（Rust側）
  - レンダラープラグインの表示名・説明・実行時文言（プラグイン自己完結の原則を維持）
- **対象外**:
  - Rustコマンドが返すエラーメッセージ本文（ダイアログの枠・見出しは対象、`詳細:` 以下の生メッセージは対象外）
  - `console.log` / `console.error` 等の開発者向けログ（日本語のまま）
  - コード内コメント・spec・ADR（開発言語は日本語のまま）
  - ja/en以外の言語追加

## ユーザーストーリー

### US-001: 言語の自動判定と切替

```
As a ユーザー
I want to OSの言語に合わせてUIが表示され、設定で明示的に切り替えもできる
So that 言語設定を意識せず使い始められる
```

**受け入れ条件**

- [x] 初期状態ではOSロケール準拠（日本語系ロケールならja、それ以外はen）
- [x] 設定パネルに「言語」選択（システム連携 / 日本語 / English）がある
- [x] 切替は再起動不要で即時反映される（表示中コンテンツのプラグイン由来文言も含む）
- [x] 言語設定は永続化され、再起動後も保持される

### US-002: ネイティブメニューの翻訳

```
As a ユーザー
I want to メニューバーもUI言語と同じ言語で表示される
So that アプリ全体で言語が統一される
```

**受け入れ条件**

- [x] 起動直後からメニューが設定言語（またはOSロケール準拠）で表示される
- [x] 設定パネルで言語を切り替えるとメニューも即時再構築される

### US-003: プラグインの多言語対応

```
As a プラグイン作者
I want to プラグインのディレクトリ内だけで自分の文言の翻訳を完結できる
So that コアの辞書ファイルに手を入れずレンダラーを追加できる
```

**受け入れ条件**

- [x] `displayName` / `description` が言語別（`{ ja, en }`）で定義でき、設定パネルが現在言語で表示する
- [x] `postRender` から現在ロケールを取得でき、実行時文言（エラー表示・ツールチップ等）を翻訳できる
- [x] plugins/README.md の作成規約が多言語対応の書き方を含む

## 未決定事項（設計フェーズで決定）

- [x] 対象言語 → ja / en
- [x] 実装方式 → 自作の型付き辞書（外部i18nライブラリは使わない）
- [x] 対象範囲 → フロントエンドUI + ネイティブメニュー（Rustエラー文言本文は対象外）
- [x] ネイティブメニューの翻訳方式 → Rust側辞書（起動時にsettings.jsonから言語を読み、切替時はコマンドで再構築）
- [x] プラグイン文言の持ち方 → プラグイン自身が言語別文言を保持（中央辞書に集約しない）
