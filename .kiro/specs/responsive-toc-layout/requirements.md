# 要求定義: レスポンシブ目次レイアウト (responsive-toc-layout)

## 背景・動機

エクスプローラーと目次を左サイドバー内で上下分割すると、どちらも利用できる縦幅が狭くなる。
横幅が十分な場合はエクスプローラー・コンテンツ・目次の3ペインへ戻し、狭い場合だけ目次をオーバーレイ表示する。

## スコープ

- **対象**: 独立した右目次ペイン、狭幅時の目次ボタンとドロワー、目次幅の永続化
- **対象外**: エクスプローラーのドロワー化、目次内容・階層生成の変更、ユーザーによるブレークポイント変更

## ユーザーストーリー

### US-001: ワイド時の3ペイン表示

**受け入れ条件**

- [x] ウィンドウ幅が1000px以上で目次表示が有効な場合、エクスプローラー・コンテンツ・目次を横3ペインで表示する
- [x] 目次はコンテンツ右側に独立して全高表示される
- [x] 目次幅をドラッグ・キーボード・ダブルクリックで変更でき、幅が永続化される
- [x] 目次表示を無効にした場合、右ペインとリサイズハンドルを表示しない

### US-002: 狭幅時の目次ドロワー

**受け入れ条件**

- [x] ウィンドウ幅が1000px未満の場合、右目次ペインを自動的に隠す
- [x] 目次表示が有効かつ見出しがある場合、コンテンツ右上にアイコンと「目次」を持つボタンを表示する
- [x] ボタンから目次を右ドロワーとして開いてもコンテンツ幅とスクロール位置を変更しない
- [x] Esc、外側クリック、閉じるボタン、見出し選択でドロワーを閉じられる
- [x] ドロワー内へフォーカスを移し、Tab循環と閉じた後のフォーカス復帰を行う
- [x] ワイド幅へ戻した場合、開いていたドロワーを閉じて右ペインへ切り替える

### US-003: 表示設定との整合

**受け入れ条件**

- [x] レスポンシブ切替は永続化された目次表示設定を変更しない
- [x] 目次表示を再度有効にしても、エクスプローラーを強制表示・強制選択しない
- [x] 印刷時は目次ペイン、目次ボタン、ドロワーを表示しない

## 未決定事項（設計フェーズで決定）

- [x] ブレークポイント: 1000px未満をドロワー表示とする
- [x] 狭幅時の起動UI: リストアイコン＋「目次」のピルボタン
- [x] ドロワー幅: 360px、ただし作業領域幅の85%を上限とする
