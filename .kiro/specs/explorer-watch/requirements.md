# 要求定義: エクスプローラーのファイル増減ウォッチ (explorer-watch)

## 背景・動機

開いているファイルの変更・削除はウォッチャーで検知しているが、エクスプローラーのツリーはファイル／フォルダの追加・削除・リネームを反映せず、フォルダを開き直すまで古い表示のままになる（2026-07-09 ユーザーフィードバック）。explorer-lazy-load により1階層の再読込が安価（十数ms）になったため、「変化があったレベルを読み直す」方式が現実的になった。

## スコープ

- **対象**: エクスプローラーに表示中のレベル（ルート＋展開中のディレクトリ）直下のファイル／フォルダの増減・リネームの検知とツリーへの自動反映
- **対象外**: 未展開・折りたたみ中のディレクトリの監視（無限に再帰しない）、ファイル内容変更のツリー上での表示（更新マーク等）、ツリーからのファイル操作（作成・削除・リネームUI）

---

## ユーザーストーリー

### US-001: 表示中レベルの自動反映

```
As a ユーザー
I want to エクスプローラーに表示されている階層でのファイル／フォルダの増減が自動で反映される
So that フォルダを開き直さなくても実際のファイル構成と一致した表示で閲覧できる
```

**受け入れ条件**

- [x] ルートまたは展開中のディレクトリ直下でファイル／フォルダを追加・削除・リネームすると、数秒以内にツリーへ反映される
- [x] 未展開・折りたたんだディレクトリは監視されない（監視対象は表示中のレベルのみ）
- [x] 反映時のフィルター・ソートは通常の読み込みと同一（gitignore考慮・拡張子フィルター・ディレクトリ優先ソート）
- [x] 反映後も、残存するディレクトリの展開状態と読み込み済みの子孫ツリーは維持される
- [x] フォルダを開き直す・別フォルダを開くと監視が張り直される（前の監視は解除）
- [x] WebViewをリロードしてもRust側の監視集合が現在の表示状態へ収束し、前の監視が残らない
- [x] 監視対象はファイル100件、Explorer 65件（ルート1件＋展開64件）を上限とする
