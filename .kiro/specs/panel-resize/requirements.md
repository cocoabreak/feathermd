# 要求定義: パネルリサイズ (panel-resize)

## 背景・動機

現在 Sidebar（`w-56` = 224px）と TOCPanel（`w-52` = 208px）は固定幅で実装されている。
core-viewer の T-012 で「3ペイン構成のドラッグリサイズ」はv2スコープとして先送りされていた。
ファイルツリーの階層が深い場合や、長い見出しを持つTOCを閲覧する場合など、ユーザーが状況に応じてパネル幅を調整したいニーズがある。

## スコープ

- **対象**: Sidebar・TOCPanel の境界をドラッグしての幅リサイズ、リサイズ幅の永続化
- **対象外**: パネルの並び替え、垂直分割、フローティング/フロートパネル化、パネルの完全な非表示位置カスタマイズ（表示/非表示トグルは実装済みのため対象外）

---

## ユーザーストーリー

### US-001: サイドバー幅のドラッグリサイズ

```
As a ユーザー
I want to Sidebarとコンテンツ領域の境界をドラッグして幅を変更する
So that ファイルツリーの表示幅を閲覧内容に合わせて調整できる
```

**受け入れ条件**

- [x] Sidebar右端にドラッグ可能なリサイズハンドルがある
- [x] ドラッグ中はマウス移動に追従してリアルタイムに幅が変わる
- [x] 最小幅・最大幅の範囲内に制限される
- [x] Sidebarが非表示のときはハンドルも表示されない

---

### US-002: TOCパネル幅のドラッグリサイズ

```
As a ユーザー
I want to TOCPanelとコンテンツ領域の境界をドラッグして幅を変更する
So that 見出しが長いドキュメントでも目次を読みやすくできる
```

**受け入れ条件**

- [x] TOCPanel左端にドラッグ可能なリサイズハンドルがある
- [x] ドラッグ中はマウス移動に追従してリアルタイムに幅が変わる
- [x] 最小幅・最大幅の範囲内に制限される
- [x] TOCPanelが非表示のときはハンドルも表示されない

---

### US-003: リサイズ幅の永続化

```
As a ユーザー
I want to 調整したパネル幅がアプリ再起動後も維持される
So that 起動のたびに幅を調整し直す必要がない
```

**受け入れ条件**

- [x] Sidebar・TOCPanelそれぞれの幅が設定として保存される（既存の `tauri-plugin-store` 永続化の仕組みを利用）
- [x] アプリ再起動後、保存した幅で復元される
- [x] 初回起動時（保存値なし）は現状のデフォルト幅（Sidebar 224px / TOC 208px）を使用する

---

## 非機能要求

| 項目 | 要求 |
|------|------|
| 応答性 | ドラッグ中のリサイズがカクつかず60fps相当で追従する |
| コンテンツ領域 | Sidebar・TOCの幅変更後もコンテンツ領域（MarkdownViewer）が最低限の幅を維持する |

---

## 未決定事項（設計フェーズで決定）

- [x] 最小幅・最大幅の具体的数値
- [x] ダブルクリックでデフォルト幅にリセットする機能の要否
- [x] リサイズハンドルの視覚表現（常時表示 vs hover時のみ表示）
