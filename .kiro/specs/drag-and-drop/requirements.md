# 要求定義: ドラッグ＆ドロップ

## 概要

Windowsのエクスプローラー等からMarkdownファイルまたはフォルダーをアプリへドロップし、ファイル選択ダイアログを経由せずに開けるようにする。

## ユーザーストーリー

### US-001: Markdownファイルを開く

**As a** ユーザー
**I want to** Markdownファイルをアプリへドロップして開く
**So that** ファイル選択ダイアログを使わずに閲覧を開始できる

#### 受け入れ条件

- [x] ウィンドウへ`.md`または`.markdown`ファイルをドロップすると、そのファイルがタブで開く
- [x] 複数のMarkdownファイルを同時にドロップすると、先頭32件までのMarkdownファイルがタブで開く
- [x] すでに開いているMarkdownファイルをドロップした場合は、既存タブがアクティブになる
- [x] Markdown以外のファイルは無視され、UIがクラッシュしない

### US-002: フォルダーを開く

**As a** ユーザー
**I want to** フォルダーをアプリへドロップして開く
**So that** フォルダー内のMarkdownファイルをエクスプローラーから閲覧できる

#### 受け入れ条件

- [x] フォルダーをドロップすると、エクスプローラーのルートとして開く
- [x] 複数フォルダーを同時にドロップした場合は、最初のフォルダーだけを開く
- [x] Markdownファイルとフォルダーを同時にドロップした場合は、最初のフォルダーと入力上限内のMarkdownファイルを開く

### US-003: 安全なパス処理

**As a** ユーザー
**I want to** ドロップしたパスだけが安全に処理される
**So that** ドラッグ＆ドロップによってファイルアクセスの信頼境界が損なわれない

#### 受け入れ条件

- [x] ドロップされたパスはファイル内容を読む前にファイル・フォルダー種別を判定する
- [x] ファイルとフォルダーは、WebViewから任意に信頼範囲を拡張できない形でRust側の信頼登録を通して開かれる
- [x] 危険なルートや存在しないパスを処理できなくても、ほかのパスの処理を継続し、UIがクラッシュしない
- [x] イベントリスナーは画面破棄時に解除される

### US-004: セッション復元との共存

**As a** ユーザー
**I want to** セッション復元確認中でも新しいファイルやフォルダーをドロップできる
**So that** 前回のセッションではなく新しい作業対象を選べる

#### 受け入れ条件

- [x] ドロップしたMarkdownファイルまたはフォルダーを1件以上正常に開いた場合、セッション復元プロンプトが閉じる
- [x] 無効なパスや非Markdownファイルしか含まれない場合、セッション復元プロンプトの状態は変わらない

## 非機能要件

- パス処理は既存のファイル・フォルダーオープン処理を再利用する
- 1件の処理失敗が同一ドロップ内の後続パスを妨げない
- Windowsを必須確認対象とする
- 1イベント当たりのパス数とMarkdownファイルの読込サイズに上限を設け、リソース枯渇を防ぐ

## スコープ外

- ドロップ中の専用オーバーレイ表示
- フォルダー内のMarkdownファイルを再帰的にすべて開く動作
- 非Markdownファイルに対するエラーダイアログ
