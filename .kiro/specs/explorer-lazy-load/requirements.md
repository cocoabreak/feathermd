# 要求定義: エクスプローラーの遅延読み込みと.gitignore考慮 (explorer-lazy-load)

> **注**: 本specは実装完了後の事後記録である。会話起点（「フォルダーを開くときに時間の掛かり具合が違う」という問題報告）で調査・設計・実装が進んだため、通常の仕様駆動フローとは順序が逆転している。経緯と意思決定の根拠を残すことを目的とする。

## 背景・動機

フォルダーを開く所要時間がフォルダーによって大きく異なる問題が報告された（2026-07-09）。調査による実測:

| フォルダ | ファイル数 | フォルダ数 | 体感 |
|---|---|---|---|
| FeatherMDリポジトリのルート | 52,049 | 4,554 | 数秒待たされる |
| 比較対象の大規模OSSリポジトリ（mermaid） | 1,785 | 265 | 即座に開く |

原因は `read_directory` コマンドが**フォルダ配下を無条件に全再帰走査**していたこと。`node_modules`（約3.1万ファイル）や Rust の `target`（約1.5万ファイル）など、ビューワーとして表示する必要のないフォルダの中身まで1エントリずつ走査し、全ツリーを一括でJSON化してWebViewへ転送していた。

## スコープ

- **対象**: エクスプローラーのフォルダオープン・ツリー展開のパフォーマンス改善、.gitignore対象エントリの表示制御（ツリー・全文検索）
- **対象外**: ツリー表示中のファイルシステム変更の自動反映（ウォッチャー連動）、除外パターンのユーザー定義

## ユーザーストーリー

### US-001: 巨大フォルダでも即座に開ける

```
As a ユーザー
I want to node_modules等を含む開発リポジトリのルートをエクスプローラーで開く
So that 配下のファイル数に関係なく待たされずにMarkdownを閲覧し始められる
```

**受け入れ条件**

- [x] フォルダを開く時間が配下の総ファイル数ではなく直下のエントリ数のみに依存する
- [x] サブフォルダの中身は展開したときに初めて読み込まれる
- [x] 読み込み中のサブフォルダにはその旨が表示される

### US-002: gitで管理しないフォルダをツリーに出さない

```
As a ユーザー
I want to .gitignore対象（node_modules等）をツリーや全文検索から除外する
So that 閲覧対象になり得ないノイズに埋もれずファイルを探せる
```

**受け入れ条件**

- [x] .gitignore対象のファイル・フォルダがツリーに表示されない（ネストした.gitignore・否定パターンも正しく解釈される）
- [x] 全文検索も.gitignore対象を検索しない
- [x] .gitignoreを考慮するかは設定でON/OFFできる（デフォルトON、永続化される）
- [x] gitリポジトリでないフォルダでは従来どおり全エントリが表示される
