# 要求定義: 安全性・安定性の強化 (stability-hardening)

## 背景・目的

0.1.0公開前の全体レビューで、既存の信頼境界に重大な破綻は見つからなかった一方、
巨大入力や多重非同期処理によるUI停止・メモリ枯渇、破棄後のイベント購読残留が確認された。

本specでは「信頼できないMarkdownを安全に閲覧できる」という既存方針を維持しつつ、
通常規模の文書での操作性を損なわない最小限の防御と負荷制御を追加する。

## スコープ

- ファイル内検索、ローカル画像、Mermaid、全文検索、Wikiリンクの負荷制御
- 非同期listener、レンダリング、状態保存の世代・直列化制御
- watcher資源の共有化
- セキュリティ境界を保ったRustモジュールの責務分割

対象外:

- Markdown編集機能
- Kroki等の外部レンダラー
- すべてのMarkdown処理を一度にWorker/Rustへ移す大規模な再実装
- Linux/macOS固有の信頼ルート方針変更
- Lightning CSSへの移行とカスタムCSS仕様変更
- Markdown画像の通信ポリシー変更

## US-001: capabilityを必要最小限にする

- [x] Tauri capabilityは実際に使用するURL open、ネイティブダイアログ、ウィンドウ操作へ限定し、WebViewのファイル書込権限を持たせない
- [x] ローカル画像は既存のAllowedRootsと最終ハンドルパス検証を維持する

## US-002: 大きい文書・多数要素でもリソースを制御できる

- [x] ファイル内正規表現検索はWebViewメインスレッドを長時間停止させない
- [x] ハイライト解除は一致件数に対して二次時間にならない
- [x] ローカル画像は重複排除し、並列数・文書単位件数・タブ切替後の反映を制御する
- [x] 印刷前Mermaid描画はbounded concurrencyと件数上限を持つ
- [x] 全文検索はblocking処理を専用executorへ分離し、latest-onlyと結果総量上限を持つ
- [x] Wikiリンク解決は同一ルートの走査結果を再利用し、毎レンダー全走査しない
- [x] 1/5/10MiB文書の実WebView計測結果を記録し、許容上限または追加対策を決定する

## US-003: 非同期処理と永続化が最新状態へ収束する

- [x] listener登録完了前に破棄されても購読が残らない
- [x] タブ切替・クローズ後に古いレンダリング結果を表示しない
- [x] Observer、DOM listener、timerは空状態とコンポーネント破棄時に解放される
- [x] tabs/settings/recent保存はkind単位でlatest-onlyに直列化し、短時間の連続変更をまとめる
- [x] 終了直前の最新状態をflushできる
- [x] キーボード再読込は最新状態のflush完了後だけ実行する

## US-004: watcherとRust責務を保守可能にする

- [x] watcherは監視パスごとの専用デバウンススレッドを作らず、共有workerで処理する
- [x] watchの同一パス登録を冪等にし、unwatchと終了時解放をテストする
- [x] `file.rs`の公開Tauri command名とセキュリティ不変条件を維持したまま責務を分割する
- [x] FSへ触れる全commandが`AllowedRoots::resolve`または明示登録経路を通ることをレビュー可能にする

## 非機能要求

- 既存のWindows主要動作を維持する
- セキュリティ検証はフロントエンドだけに依存しない
- 負荷制限は定数化し、境界値テストを持つ
- 既存の104件のフロントテスト、53件のRustテストを回帰させない
- 変更後に設計レビューとセキュリティレビューを実施する
