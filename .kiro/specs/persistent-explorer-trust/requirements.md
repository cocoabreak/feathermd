# 要求定義: Explorerルートの限定永続信頼

## 目的

前回終了時のExplorerルートを再度開く際のアクセス確認を省きつつ、最近使った別フォルダーやWebView状態を無条件の認可根拠にしない。

## 受け入れ条件

- [x] 永続信頼するルートは最後に明示承認されたExplorerルート1件だけとする。
- [x] ネイティブフォルダー選択、またはRustネイティブ確認で承認されたフォルダーだけが永続信頼を更新できる。
- [x] 再起動時に保存ルートをcanonicalizeし、存在・ディレクトリ種別・危険ルート拒否を再検証する。
- [x] 保存ルートが前回セッションのExplorerルートと一致すれば確認なしで復元できる。
- [x] 最近使った別フォルダーは従来どおりネイティブ確認を表示する。
- [x] WebViewは永続信頼データの保存先を指定・読み書きできない。
- [x] 既存settings/tabs/recentの保存形式とデータを維持する。

## 対象外

- 複数の永続信頼ルート一覧
- OS資格情報ストアによる暗号化
- D&D・CLI入力だけを根拠とする永続信頼
