# 要求定義: 文字数・読了時間の表示 (reading-stats)

## 背景・動機

表示中のMarkdownファイルの分量感（文字数）と、読むのにどれくらいかかりそうか（読了時間の目安）をステータスバーで確認できるようにしたい。

## スコープ

- **対象**: 現在アクティブなタブの本文（コードブロックを除く）の文字数/単語数、および推定読了時間の表示
- **対象外**: 正確な読了時間の予測（あくまで目安）、多言語の厳密な言語判定（CJK比率による簡易ヒューリスティックのみ）、ユーザーごとの読書速度設定

---

## ユーザーストーリー

### US-001: 文字数・読了時間の表示

```
As a ユーザー
I want to 表示中のファイルの文字数と推定読了時間を確認する
So that ファイルの分量感を把握できる
```

**受け入れ条件**

- [x] ステータスバーに、現在表示中のファイルの文字数（または単語数）が表示される
- [x] 同じ箇所に推定読了時間（「約n分」）が表示される
- [x] コードブロックの内容は文字数・読了時間の計算に含まれない
- [x] ファイルが読み込まれていない場合（タブなし等）は表示されない

---

### US-002: 言語に応じた計測方法の切り替え

```
As a ユーザー
I want to 日本語中心の文章でも英語中心の文章でも妥当な読了時間が出る
So that 言語によって目安が大きく外れない
```

**受け入れ条件**

- [x] 本文中のCJK文字（漢字・ひらがな・カタカナ）の比率が高い場合、文字数ベース（目安: 500文字/分）で計算する
- [x] CJK比率が低い場合、単語数ベース（目安: 200 words/分）で計算する

---

## 非機能要求

| 項目 | 要求 |
|------|------|
| パフォーマンス | 大きいファイルでも、既存のTOC構築・画像パス変換等と同等程度の軽量な後処理で完結する（DOMの複製等、重い処理をしない） |

## 未決定事項（設計フェーズで決定）

- [x] CJK比率のしきい値
- [x] ステータスバー内の表示位置・フォーマット
