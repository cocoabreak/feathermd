# 要求定義: 複数インスタンス起動の抑止 (single-instance)

## 背景・動機

現状、ショートカット等から複数回起動すると、その都度新しいウィンドウが独立して立ち上がる。各ウィンドウはタブ・設定状態を共有しないため、「開いていたファイルがどちらのウィンドウか分からない」といった混乱を招く。

## スコープ

- **対象**: 2回目以降の起動を検知し、新規ウィンドウを作らず既存ウィンドウをフォーカスする
- **初期スコープ外**: 2回目起動時に渡されたファイルパスを既存ウィンドウで開く連携。本spec完成後、`.kiro/specs/windows-shell-integration/`でCLI入力と合わせて実装する。

---

## ユーザーストーリー

### US-001: 二重起動時の既存ウィンドウフォーカス

```
As a ユーザー
I want to アプリを誤って複数回起動しても新しいウィンドウが増えない
So that どのウィンドウが最新の状態か迷わずに済む
```

**受け入れ条件**

- [x] アプリ起動中に再度アプリを起動すると、新しいウィンドウは作られない
- [x] 2回目の起動操作をすると、既存のウィンドウが最前面にフォーカスされる（最小化されていれば復元される）
- [x] 通常の単一起動時の挙動（起動・終了）に回帰がない

---

## 非機能要求

なし（既存のウィンドウ管理と同程度の軽量な対応で十分）

## 未決定事項（設計フェーズで決定）

- [x] 対象OS範囲（Windows/Linux/macOSすべてに適用するか）
- [x] フォーカス対象ウィンドウの特定方法（ウィンドウlabelの扱い）
- [x] 最小化されている場合の復元方法
