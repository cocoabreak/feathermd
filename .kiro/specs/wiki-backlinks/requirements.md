# 要求定義: Wikiリンクのバックリンク

## 背景・目的

Wikiリンクで結ばれたMarkdown群を閲覧するとき、現在文書から参照先へ進むだけでなく、現在文書を参照している文書へ戻れるようにする。既存のDocumentSource、AllowedRoots、Wikiリンク解決規則を維持し、Native FSとZIPを同じUIで扱う。

## スコープ

- 左サイドバーの「バックリンク」タブ
- 現在文書を参照するWikiリンクの参照元文書・相対パス・参照数の一覧
- NativeSourceとZipSourceの遅延索引構築と再利用
- 明示的な再読み込みと既知のファイル・ディレクトリ変更後の再構築
- 件数・読込量・解決処理量の上限と省略表示

## 対象外

- 通常のMarkdownリンクによる参照の収集
- 参照箇所の本文プレビュー、行番号表示、該当行へのジャンプ
- バックリンク索引のディスク永続化
- OS全体や別DocumentSourceを横断する索引

## US-001: 現在文書のバックリンクを確認する

- [x] 左サイドバーの3番目のタブからバックリンク一覧を開ける
- [x] 有効な現在文書がありWikiリンクプラグインが有効な場合だけ索引を遅延構築する
- [x] 各結果に参照元のファイル名、相対パス、同一文書からの参照数を表示する
- [x] 結果がない場合、読込中、失敗、省略ありを区別して表示する
- [x] 再読み込みボタンでキャッシュを破棄して再構築できる

## US-002: 参照元へ移動する

- [x] 結果を選択すると同じDocumentSourceの参照元文書を既存の`openSourceMarkdown`経路で開く
- [x] 読み込み中にタブまたはSourceが変わった場合、古い結果で現在状態を上書きしない
- [x] Native FSとZIPのどちらでも同じ操作で移動できる

## US-003: 安全に索引を構築する

- [x] Rust側でSourceRegistryとAllowedRootsを通して候補列挙・本文読み込みを行う
- [x] 既存Wikiリンクと同じターゲット解決規則で、重複ファイル名がある場合も実際の遷移先だけをバックリンクとする
- [x] コードスパン・コードブロック内の`[[...]]`は参照として数えない
- [x] 対象文書数、展開後総読込量、ZIP圧縮データ量、リンク数、解決処理量に上限を設ける
- [x] 上限到達や個別ファイルの読込失敗は`truncated`として返し、無制限処理や信頼境界の迂回を行わない

## 非機能要求

- 新しい依存関係、Tauri capability、永続化ファイルを追加しない
- バックリンクタブを開かない限り本文走査を行わない
- 索引はSource ID、generation、隠しファイル設定、gitignore設定をキーにメモリ内で最大4件だけ保持する
- キャッシュは一定時間で失効し、既知のwatcherイベント後は次回表示時に再構築する
