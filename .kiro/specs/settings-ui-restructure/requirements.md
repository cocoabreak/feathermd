# 要求定義: 設定画面のカテゴリ再構成 (settings-ui-restructure)

## 背景・動機

設定項目の増加により単一の縦長リストから目的の項目を探しにくくなった。保存形式や各設定の動作を変えず、カテゴリ単位で移動できる設定画面へ再構成する。

## スコープ

- **対象**: 設定ダイアログのカテゴリナビゲーション、レスポンシブ配置、セクション折りたたみ、カテゴリ移動のアクセシビリティ、既存設定項目の再配置
- **対象外**: 新しい設定値の追加、設定ファイル形式の変更、ブレークポイントのユーザー設定化、About／ライセンス表示

## 受け入れ条件

- [x] 「表示」「ファイル」「プライバシー」「レンダラー」の4カテゴリから目的の設定へ移動できる
- [x] 広い表示ではカテゴリナビゲーションを左側、狭い表示では内容幅を圧迫しない横並びで表示する
- [x] 各カテゴリ内の複数セクションを個別に折りたたみ・展開できる
- [x] 選択中カテゴリを視覚表示と`aria-current`の両方で識別でき、キーボードだけでカテゴリを切り替えられる
- [x] ダイアログのフォーカストラップ、Escape終了、外側クリック終了を維持する
- [x] 全既存設定がいずれかのカテゴリに存在し、変更時の即時反映・自動保存を維持する
- [x] 設定ストアの永続化スキーマを変更しない

## カテゴリ割り当て

| カテゴリ | セクション | 設定 |
|---|---|---|
| 表示 | アプリ、コード、レイアウト、カスタマイズ | 言語、テーマ、コードテーマ、行番号、サイドバー、目次、カスタムCSS |
| ファイル | エクスプローラー、外部連携 | 隠しファイル、`.gitignore`、外部エディター |
| プライバシー | 外部コンテンツ | 外部HTTPS画像ポリシー |
| レンダラー | レンダラープラグイン | Mermaid、KaTeX、Wiki Links等の有効・無効 |

## 未決定事項（設計フェーズで決定）

- [x] カテゴリ数: 4カテゴリ
- [x] 狭幅時のナビゲーション: 横スクロール可能な上部ナビゲーション
- [x] 折りたたみ状態: ダイアログ内だけの一時状態とし、永続化しない
