# 技術設計: 設定画面のカテゴリ再構成 (settings-ui-restructure)

## ステータス

完了

## 1. 構造

- `SettingsPanel.svelte`のダイアログを、ヘッダー、カテゴリナビゲーション、カテゴリ内容、フッターの4領域へ分ける
- `activeCategory`はコンポーネントローカル状態とし、ダイアログを開くたび「表示」を初期値にする
- カテゴリは通常の`button`によるナビゲーションとし、選択中項目へ`aria-current="page"`を付ける。タブ固有の矢印キー操作を要求するARIA tabパターンは使わない
- カテゴリ内容には見出しを置き、ナビゲーションと同じ名称を明示する

## 2. レスポンシブ配置

- ダイアログ最大幅を約56remへ広げ、内容領域を`sm`以上で左右配置する
- `sm`未満ではナビゲーションを上部の横並びへ切り替え、`overflow-x-auto`でカテゴリを欠落させない
- 内容領域だけをスクロールさせ、ヘッダー、ナビゲーション、フッターの操作性を維持する

## 3. セクション折りたたみ

- ネイティブ`details`／`summary`を使い、追加JavaScriptなしでキーボード操作と開閉状態を提供する
- 初期状態は全セクションを展開し、現在と同じ発見性を維持する
- 折りたたみ状態は設定値ではないため保存しない

## 4. 既存動作の維持

- `settingsStore`と`saveSettings()`の呼び出しは既存のままとし、設定スキーマを変更しない
- 言語変更時のネイティブメニュー再構築、`.gitignore`変更時のツリー再読込、カスタムCSSの適用、外部エディターのRust側認可を維持する
- 既存のfocus trapとダイアログ終了操作を維持する

## 5. テスト

- 初期カテゴリ、全カテゴリ名、カテゴリ切り替え、`aria-current`をコンポーネントテストで確認する
- 各カテゴリの代表設定が正しいカテゴリで表示されることを確認する
- format、lint、Svelte check、Vitest、build、Rust検査、実WebViewでの狭幅／広幅確認を行う
