# 要求定義: 大容量Markdownのレンダリング前警告

## 背景

実WebView計測では1MiB文書が約4.4秒、5MiB文書が約86秒、10MiB文書は180秒以内に描画完了せず、WebViewも応答不能になった。既存の10MiB読込上限だけでは、レンダリング開始後の操作不能を防げない。

## US-001: 大容量文書を明示確認なしにレンダリングしない

- [x] 5MiB以上10MiB以下のMarkdownを開くと、本文をレンダリングする前に警告を表示する
- [x] 警告にはファイル名と、処理に長時間かかりアプリが応答しなくなる可能性を表示する
- [x] ユーザーが拒否した場合、タブ・最近使ったファイル・watcher・本文キャッシュを変更しない
- [x] ユーザーが承認した場合のみ、従来どおりタブを開いてレンダリングする

## US-002: すべてのオープン経路で一貫して保護する

- [x] ファイル選択、Explorer、最近使ったファイル、リンク、全文検索、D&D、CLI、セッション復元に同じ判定を適用する
- [x] 承認済みパスはアプリ実行中だけ記憶し、同じファイルの監視更新では再確認しない
- [x] アプリ再起動後は大容量ファイルを再度確認する

## US-003: 既存の信頼境界と読込上限を維持する

- [x] サイズはRust側でオープン済みファイルのmetadataから取得する
- [x] AllowedRoots、最終ハンドルパス検証、Markdown拡張子制限、10MiB上限を維持する
- [x] 10MiBを超えるファイルは従来どおり拒否し、警告による上限回避を許さない

## 非機能要求

- 警告対象の判定はレンダリング開始前に完了する
- 閾値は定数化し、RustとTypeScriptの不整合を作らない
- 日本語・英語の表示に対応する
