# 要求定義: タブ状態の永続化・復元 (session-restore)

## 背景・動機

現状、開いているタブの一覧は完全にメモリ内状態のみで、ディスクへの永続化を一切行っていない。そのため、WebViewの右クリックメニューからの「再読み込み」やアプリの再起動を行うと、開いていたタブが全て消えてしまう。

本アプリはタブを複数持つ設計のため、開いていたタブ一覧（複数）・ピン留め状態・アクティブタブを丸ごと復元する。加えて、エクスプローラーで開いていたフォルダーのルートパスもタブと同様に消えてしまうことが判明したため、同じ仕組みで復元対象に含める。

WebView自体の右クリックメニュー「再読み込み」を無効化する対応も検討したが、`Ctrl+R`等の他経路からのリロードには効かないため対象外とし、状態の永続化・復元のみで対応する。

当初は起動時に常に自動復元する設計だったが、`Ctrl+R`/`F5`のようなリロード（Rustプロセスは継続する）と、アプリ本来の再起動（プロセスの初回起動）を同列に扱うのは過剰との判断に至った。ブラウザ（Edge等）を参考に、リロードは自動復元を維持しつつ、アプリ再起動時のみ「復元しますか？」の確認をトースト形式で挟む方式に変更した。

## スコープ

- **対象**: 開いているタブの一覧（ファイルパス・ピン留め状態）・アクティブタブ・エクスプローラーのルートフォルダーパスの永続化。`Ctrl+R`/`F5`リロード時の自動復元。アプリ再起動時の復元確認トースト
- **対象外**: スクロール位置・TOCの開閉状態・エクスプローラーの展開中サブフォルダ（`expandedDirs`）等、細かいUI状態の復元。WebView自体の右クリックメニューの無効化。復元時にファイル/フォルダが見つからない場合のエラー表示（黙ってスキップする）

## ユーザーストーリー

### US-001: タブ状態の自動保存

```
As a ユーザー
I want to タブの追加・削除・切り替え・ピン留め変更が自動的に保存される
So that いつリロード・再起動しても直前の状態が失われない
```

**受け入れ条件**

- [x] タブを開く・閉じる・切り替える・ピン留めする、いずれの操作後も現在のタブ一覧とアクティブタブが保存される
- [x] エクスプローラーで開くフォルダーを変更した場合も、そのルートフォルダーパスが保存される
- [x] 保存は自動（ユーザー操作不要）で、既存の設定保存等と同様にファイル入出力の失敗はコンソール警告に留め、UI操作をブロックしない

---

### US-002: リロード時のタブ自動復元

```
As a ユーザー
I want to WebViewの右クリック「再読み込み」やCtrl+R/F5をしても前回開いていたタブが自動で開き直される
So that リロードのたびにファイルを開き直さずに済む
```

**受け入れ条件**

- [x] `Ctrl+R`/`F5`等によるリロード時、保存されていたタブ一覧が同じ順序で自動的に開き直される（確認なし）
- [x] 保存されていたピン留め状態が復元される
- [x] 保存されていたアクティブタブが復元後もアクティブになる
- [x] 保存されていたルートフォルダーがあれば、エクスプローラーにそのフォルダーが自動的に開き直される（展開中サブフォルダの状態までは復元しない）
- [x] 保存されていたパスのファイル/フォルダーが既に存在しない場合、そのタブ/フォルダー復元は黙ってスキップされる（エラー表示なし）
- [x] 保存済みタブ・フォルダーがない場合は従来通りの空状態画面になる

---

### US-003: アプリ再起動時の復元確認

```
As a ユーザー
I want to アプリを再起動した際、前回のセッションを復元するかどうかを選べる
So that 意図せず前回の続きを開かされたり、逆に前回の作業が黙って失われたりしない
```

**受け入れ条件**

- [x] アプリ本来の起動（プロセスの初回起動）時は自動復元せず、保存済みタブが1件以上、またはルートフォルダーが保存されている場合のみ画面下部にトースト（「前回のセッションを復元しますか？」＋「復元する」「破棄する」ボタン）を表示する
- [x] 保存済みタブ・フォルダーがともにない場合はトーストを表示せず、従来通りの空状態画面になる
- [x] 「復元する」を選ぶと、US-002と同じ内容でタブ・フォルダーが開き直される
- [x] 「破棄する」を選ぶと、保存済みセッション情報（`tabs.json`、ルートフォルダー含む）を即座に破棄し、空の状態で使い始める
- [x] トースト表示中に新規でタブを開く等の操作をしても、保存済みセッション情報を上書きしない（判断前に消えないようにする）

---

## 非機能要求

なし（既存の設定・最近使ったファイルの永続化と同程度の軽量な対応で十分）

## 未決定事項（設計フェーズで決定）

- [x] 永続化ファイル（`settings.json`に統合するか別ファイルにするか）
- [x] 保存トリガーの実装方式（各操作箇所で明示的に呼ぶか、リアクティブな監視にするか）
- [x] 復元時にファイル読み込みが失敗した場合の扱い（タブ自体を復元しないか、エラー状態のタブとして残すか）
