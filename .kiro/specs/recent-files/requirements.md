# 要求定義: 最近開いたファイル・フォルダー (recent-files)

## 背景・動機

ファイル/フォルダをOSのダイアログから毎回選び直すのは手間なため、直近に開いたものを一覧から素早く再度開けるようにしたい。

## スコープ

- **対象**: 最近開いたMarkdownファイル・フォルダーをそれぞれ最大10件記録し、アプリ再起動後も保持する。何もタブ/フォルダが開かれていない空状態の画面に一覧を表示し、クリックで再度開く
- **対象外**: 一覧からの個別削除・クリア操作（個別削除はのちに `ui-refinements` specで実現）、存在しなくなったパスの自動除去、タブの状態そのものの永続化（セッション復元は別スコープ、`.kiro/backlog.md`参照）

---

## ユーザーストーリー

### US-001: 最近開いたファイルの表示・再オープン

```
As a ユーザー
I want to 最近開いたMarkdownファイルを一覧から選んで開く
So that 毎回ファイルダイアログを操作しなくて済む
```

**受け入れ条件**

- [x] ファイルが1つも開かれていない状態で、最近開いたファイルがあれば一覧が表示される
- [x] 一覧の項目をクリックすると、そのファイルがタブで開かれる
- [x] ファイルオープン（ダイアログ・サイドバーのファイルツリー・本文中のリンク経由いずれも）のたびに、最近開いたファイル一覧の先頭に追加される
- [x] 同じパスが複数回開かれても一覧内で重複せず、最新の1件として先頭に移動する
- [x] 一覧は最大10件まで保持する

---

### US-002: 最近開いたフォルダーの表示・再オープン

```
As a ユーザー
I want to 最近開いたフォルダーを一覧から選んで開く
So that 毎回フォルダダイアログを操作しなくて済む
```

**受け入れ条件**

- [x] フォルダが開かれていない状態（サイドバーの空状態）で、最近開いたフォルダーがあれば一覧が表示される
- [x] 一覧の項目をクリックすると、そのフォルダーがエクスプローラーのルートとして開かれる
- [x] フォルダオープンのたびに一覧の先頭に追加され、重複排除・最大10件保持はUS-001と同様

---

### US-003: 再起動後も一覧が保持される

```
As a ユーザー
I want to アプリを再起動しても最近開いた一覧が残っている
So that 前回の続きからすぐに作業を再開できる
```

**受け入れ条件**

- [x] アプリ起動時に、保存済みの最近開いた一覧が復元される

---
