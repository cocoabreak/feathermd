# 要求定義: Windowsシェル統合とCLI入力

## 背景・目的

FeatherMDをWindowsのMarkdownビューワーとして日常的に使えるよう、Explorerの右クリックメニューとコマンドラインから文書を開けるようにする。既定アプリは変更せず、ユーザーが設定画面から明示的に登録・解除できる補助動詞として提供する。

既存コードには初回起動引数、single-instance経由の2回目起動引数、ウィンドウへのドラッグ＆ドロップを共通処理する基盤がある。本specではその経路を正式な機能として完成させ、相対パスや設定UIを含めて検証する。

## スコープ

- Windows Explorerで`.md` / `.markdown`を右クリックした際の「FeatherMDで開く」
- 設定画面からのユーザー単位の登録・解除
- 初回起動および起動済みプロセスへのCLIパス入力
- CLIの相対パス解決、入力数制限、既存の信頼境界との統合

## 対象外

- `.md` / `.markdown`の既定アプリをFeatherMDへ強制変更すること
- Windows 11の新しい短縮コンテキストメニューへ直接表示するCOMシェル拡張
- システム全ユーザーへの登録、管理者権限の要求
- CLIからの標準出力、Markdown変換、編集操作
- Windows以外のOS固有シェル統合

## US-001: 右クリックメニューの登録・解除

As a ユーザー
I want to 設定画面から「FeatherMDで開く」を登録・解除する
So that 既定アプリを変えず、必要なときだけExplorerからFeatherMDで閲覧できる

### 受け入れ条件

- [x] 初期状態では自動登録しない
- [x] Windowsの設定画面「ファイル」に登録トグルを表示する
- [x] ONにすると現在ユーザーの`.md` / `.markdown`用右クリック動詞を登録する
- [x] OFFにするとFeatherMD自身が所有する動詞キーだけを削除する
- [x] 既定アプリおよび他アプリの関連付けを変更・削除しない
- [x] 管理者権限を要求しない
- [x] 登録状態はsettings.jsonの推測値ではなく、現在の実行ファイルを指すレジストリ値から判定する
- [x] 登録・解除後にExplorerへ関連付け変更を通知する
- [x] Windows 11では「その他のオプションを確認」側に表示される可能性をUIで説明する
- [x] Windows以外では操作を表示しない

## US-002: CLIからパスを開く

As a ユーザー
I want to `FeatherMD.exe <path>...`で文書を開く
So that ターミナル、Explorerの右クリック、ショートカット等から同じ閲覧経路を利用できる

### 受け入れ条件

- [x] 初回起動引数のMarkdownファイル、フォルダー、ZIPを既存の画面操作と同じ経路で開く
- [x] 起動済みの場合は新規ウィンドウを作らず、既存ウィンドウを復元・フォーカスして開く
- [x] 相対パスは各起動要求を発行したプロセスのカレントディレクトリを基準に絶対化する
- [x] 複数入力を扱い、Rust側・フロント側とも最大32件に制限する
- [x] listener登録前やWebView再読み込み中の入力をRust側で保持し、次の回収時に開く
- [x] 存在しないパス、対象外拡張子、2つ目以降のフォルダー/ZIPは既存方針どおり安全に無視する
- [x] CLI引数はRustが直接受け取ったOS入力としてのみ`AllowedRoots`へ登録し、WebViewイベントを認可信頼の根拠にしない
- [x] Markdownのサイズ確認、ZIPの展開後サイズ確認、危険ルート制限等の既存保護を迂回しない

## US-003: ドラッグ＆ドロップとの整合

As a ユーザー
I want to ファイルをFeatherMDへドロップして開く
So that Explorerから直感的に文書を閲覧できる

### 受け入れ条件

- [x] 開いているFeatherMDウィンドウへのドロップは既存のTauri drag-drop経路で動作する
- [x] 実行ファイルまたはショートカットへのドロップでWindowsがファイルパスをCLI引数として渡した場合も開ける
- [x] DnDとCLIはフロント側のパス処理を共有するが、Rust側の信頼登録は各OS入力イベントから直接行う

## 非機能要求

- レジストリへ書くコマンド文字列は実行ファイルパスと`%1`を個別に引用し、空白を含むパスでも正しく動作する
- 部分的な登録失敗時は作成済みのFeatherMD動詞をロールバックし、中途半端なON状態を残さない
- レジストリ操作の失敗を握りつぶさず、設定画面へエラーを返す
- 新しい汎用レジストリ書き込みIPCは公開せず、固定されたFeatherMD動詞のON/OFFだけを公開する

## 未決定事項（設計フェーズで決定）

- [x] 既定アプリ登録か補助動詞か → 補助動詞のみ
- [x] 登録範囲 → `HKEY_CURRENT_USER`の現在ユーザーのみ
- [x] 対象拡張子 → `.md` / `.markdown`
- [x] Windows 11短縮メニュー対応 → COM拡張は対象外、従来型静的動詞
- [x] CLI対象 → Markdown、フォルダー、ZIP
- [x] 初期値 → OFF（明示操作時のみ登録）
