# 要求定義: コアビューワー (core-viewer)

## 背景・動機

VS CodeなどのエディタはMarkdownプレビューが可能だが多機能ゆえに重い。
ブラウザ拡張でも閲覧できるが、Mermaidなどの図やコードブロックの表示に不満がある。
**軽量かつMermaid・TeX・Krokiなどのリッチなレンダリングに対応したMarkdown専用ビューワー**が必要。

## スコープ

- **対象**: ローカルファイルシステム上のMarkdownファイルの閲覧（ビューワー専用、編集機能は対象外）
- **実装基盤**: TauriまたはWails（WebView活用のデスクトップアプリ。技術選定は設計フェーズで決定）

---

## ユーザーストーリー

### US-001: 単一ファイルのオープン

```
As a ユーザー
I want to ファイル選択ダイアログから単一のMarkdownファイルを開く
So that 素早く1つのファイルを閲覧できる
```

**受け入れ条件**

- [x] ファイル選択ダイアログが開き、`.md` / `.markdown` ファイルを選択できる
- [x] 選択したファイルがレンダリングされてビューワー内に表示される
- [x] 無効なファイル（非Markdown）を選択した場合、適切なエラーメッセージが表示される

---

### US-002: ディレクトリエクスプローラー

```
As a ユーザー
I want to ディレクトリを指定してサイドパネルにファイルツリーを表示する
So that ディレクトリ内のMarkdownファイルを一覧・選択できる
```

**受け入れ条件**

- [x] ディレクトリ選択ダイアログでフォルダを指定できる
- [x] サイドパネルにVS Code風のファイルエクスプローラーが表示される
- [x] ツリーはディレクトリ構造を再帰的に表示し、フォルダの展開・折りたたみができる
- [x] 表示対象は `.md` / `.markdown` ファイルのみ（他拡張子は非表示）
- [x] 隠しファイル・ディレクトリ（`.`始まり）はデフォルト非表示。設定でトグルON可能
- [x] ファイルをクリックするとビューワーに表示される

---

### US-003: タブによる複数ファイルの同時表示

```
As a ユーザー
I want to 複数のMarkdownファイルをタブで開く
So that ファイル間をすばやく切り替えて閲覧できる
```

**受け入れ条件**

- [x] ファイルを開くたびに新しいタブが追加される（同一ファイルを開こうとした場合は既存タブをアクティブにする）
- [x] タブをクリックするとそのファイルのビューに切り替わる
- [x] タブに閉じるボタンがあり、個別にタブを閉じられる
- [x] タブ数に上限はなく、増えた場合は横スクロールで対応する
- [x] タブにはファイル名が表示される

---

### US-004: ファイル更新の自動検知と再描画

```
As a ユーザー
I want to ファイルが外部で変更された際に自動でプレビューが更新される
So that 常に最新の内容を確認できる
```

**受け入れ条件**

- [x] 開いているファイルへの変更をファイルシステムレベルで監視する
- [x] ファイル変更検知後、ユーザー操作なしに自動でプレビューを再描画する
- [x] 再描画後、スクロール位置は可能な限り維持される
- [x] ファイルが削除された場合、タブにその旨を表示する

---

### US-005: リンクのナビゲーション

```
As a ユーザー
I want to Markdown内のリンクをクリックしたとき、リンク種別に応じた動作をする
So that ビューワー内外をシームレスに移動できる
```

**受け入れ条件**

- [x] ローカルファイルへのリンク（相対パスまたは絶対パス）はビューワー内で開く
- [x] `https://` / `http://` 始まりのリンクはOSデフォルトブラウザで開く
- [x] 同一ファイル内のアンカーリンク（`#heading`）は該当セクションへスクロールする
- [x] リンク先のMarkdownファイルが存在しない場合、エラーを表示する

---

### US-006: 拡張レンダラー（プラグイン機構）

```
As a ユーザー
I want to Mermaid・Kroki・TeXなどのレンダラーを有効化・設定できる
So that 図・数式を含むMarkdownを正しく表示できる
```

**背景**
レンダラーには実行方式が異なる複数の種別が存在する：
- **JS内蔵型**: WebView内で完結（Mermaid、KaTeX/MathJax など）
- **外部プロセス型**: ローカルのプロセス・コンテナを起動して結果を取得（Kroki ローカル、PlantUML JAR など）
- **クラウドAPI型**: 外部サービスのAPIを呼び出して結果を取得（Kroki クラウド、PlantUML Online など）

**Krokiについて**
Kroki（https://kroki.io/）はPlantUML・Mermaid・GraphViz・C4・Excalidraw・Ditaaなど多数の図形式を統一的に扱えるツール。
ローカルコンテナ（Docker）またはクラウドAPI（kroki.io）の両方で動作する。

**PlantUMLの優先度**
PlantUMLは将来対応とし、Krokiのバックエンドとして包含される想定。v1では対象外。

**受け入れ条件**

**初期スコープ（v1）**
- [x] Mermaid、TeX（KaTeX または MathJax）をJS内蔵型として標準バンドルする
- [x] 各レンダラーの有効/無効を設定UIで切り替えられる
- [x] レンダリングに失敗したブロックはエラー表示し、他のコンテンツ表示に影響しない

**将来スコープ（v2以降、設計時に考慮するが実装はしない）**
- [ ] Krokiを外部プロセス型またはクラウドAPI型として追加できる
- [ ] 外部プロセス型・クラウドAPI型それぞれで接続先（エンドポイント・パス）を設定できる
- [ ] アプリ再ビルドなしにプラグインを追加できる機構
- [ ] プラグインごとにON/OFFを設定で管理できる

**設計上の制約**
- v1においても、JS内蔵型・外部プロセス型・クラウドAPI型の3種別を考慮した抽象インターフェースを設けること
- 外部プロセス型はプロセス起動・通信・ライフサイクル管理の拡張ポイントを持つこと

---

### US-007: 目次（TOC）機能

```
As a ユーザー
I want to 見出しから自動生成された目次を表示し、コンテンツとの双方向スクロール同期ができる
So that 長いドキュメントを効率よく閲覧・ナビゲートできる
```

**受け入れ条件**

- [x] MarkdownのH1〜H6見出しから目次を自動生成してパネルに表示する
- [x] 目次の項目をクリックすると、コンテンツの該当見出しへスムーズスクロールする
- [x] コンテンツをスクロールすると、ビューポートに表示中の見出しに対応する目次項目がハイライトされる（スクロール追随）
- [x] 見出しがないMarkdownの場合、目次パネルは非表示またはその旨を表示する
- [x] 目次パネルは表示/非表示をトグルできる

---

## 非機能要求

| 項目 | 要求 |
|------|------|
| 起動速度 | アプリ起動からファイル表示まで3秒以内 |
| レスポンス | ファイル切り替え・スクロールが60fps相当でスムーズ |
| 対応OS | **Windows: Must** / Linux: 対応したい（Nice to have） / macOS: 最低優先度 |
| ファイルサイズ | 少なくとも1MB程度のMarkdownファイルを問題なく表示できる |
| 文字選択 | 原則としてMarkdownのプレビュー（レンダリング・ソース・セーフモード）と入力・編集可能要素だけを選択可能とし、ウェルカム・読み込み表示を含むアプリUIは選択不可とする |
