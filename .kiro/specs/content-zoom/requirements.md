# 要求定義: コンテンツズーム (content-zoom)

## 背景・動機

現状、Markdownプレビューの文字サイズは固定である。ディスプレイの解像度・DPIやユーザーの視力によって読みやすいサイズは異なるため、コンテンツ領域の文字サイズを拡大縮小できるようにする。

画像・Mermaid個別のズーム（クリックでの拡大表示等）は別機能（別spec）として切り分け、本specの対象外とする。

## スコープ

- **対象**: コンテンツ領域（`.markdown-body`、Markdownのレンダリング結果）の文字サイズの拡大縮小、ズーム率の永続化
- **対象外**: Toolbar・Sidebar・TOCPanel・StatusBar等UI本体の拡大縮小、画像/Mermaid個別のズーム（別spec）、タブ・ファイルごとに異なるズーム率を保持すること（v1ではアプリ全体で単一のズーム率）

---

## ユーザーストーリー

### US-001: キーボードでのズーム操作

```
As a ユーザー
I want to キーボードショートカットでコンテンツの文字サイズを拡大縮小する
So that 読みやすいサイズに素早く調整できる
```

**受け入れ条件**

- [x] `Ctrl++`（`Ctrl+=`を含む）でズームインする
- [x] `Ctrl+-` でズームアウトする
- [x] `Ctrl+0` でデフォルト（100%）にリセットする
- [x] 最小・最大のズーム率でそれ以上拡大/縮小されない（クランプされる）
- [x] ズーム操作はコンテンツ領域の文字サイズのみに影響し、Toolbar等UI本体のサイズは変わらない

---

### US-002: ズーム率の表示

```
As a ユーザー
I want to 現在のズーム率を確認する
So that どの程度拡大縮小されているか把握できる
```

**受け入れ条件**

- [x] StatusBarに現在のズーム率（例: `100%`）が表示される
- [x] ズーム率をクリックすると100%にリセットされる

---

### US-003: ズーム率の永続化

```
As a ユーザー
I want to 設定したズーム率がアプリ再起動後も維持される
So that 起動のたびに調整し直す必要がない
```

**受け入れ条件**

- [x] ズーム率が設定として保存される（既存の`tauri-plugin-store`永続化の仕組みを利用）
- [x] アプリ再起動後、保存したズーム率で復元される
- [x] 初回起動時（保存値なし）は100%を使用する

---

## 非機能要求

| 項目 | 要求 |
|------|------|
| 表示崩れ | 最小・最大ズーム率でもレイアウトが崩れない（横スクロールが必要になる場合は許容） |

---

## 未決定事項（設計フェーズで決定）

- [x] 最小・最大ズーム率、1段階あたりの増減幅の具体的数値
- [x] `Ctrl+マウスホイール`でのズーム操作の要否
- [x] StatusBarでの表示・操作のUI（`-`/`+`ボタンの要否）
