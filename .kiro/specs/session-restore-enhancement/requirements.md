# 要求定義: セッションUI状態の復元強化 (session-restore-enhancement)

## 背景・動機

既存の `session-restore` では、タブ一覧・アクティブタブ・ピン留め・Explorerのルートを保存・復元できる。一方、タブごとのスクロール位置、Explorerで展開していたサブフォルダー、ファイル内検索の状態はプロセス終了時に失われるため、復元後に閲覧位置と作業文脈を手作業で探し直す必要がある。

既存の復元確認フロー（リロード時は自動復元、完全再起動時は「復元する」「破棄する」を選択）を維持したまま、これらのUI状態も同じセッションに含める。

## スコープ

- **対象**: タブごとの表示モード別スクロール位置、Explorerの展開中ディレクトリ、ファイル内検索の開閉・検索語・正規表現設定、再読込直前の最新状態保存
- **対象外**: 検索結果・マッチ件数・現在マッチ、TOCの現在見出し、コンパクト表示時のTOCドロワー開閉、旧開発版 `tabs.json` との互換・移行
- TOCパネルの表示設定は `settings.json` で既に永続化されている。現在見出しは復元したスクロール位置から再計算する

---

## ユーザーストーリー

### US-001: 閲覧位置の復元

As a ユーザー  
I want to タブごとの閲覧位置が復元される  
So that 前回読んでいた場所から再開できる

**受け入れ条件**

- [x] 各タブのレンダリング表示・ソース表示・セーフモード表示のスクロール位置が個別に保存される
- [x] リロードまたはセッション復元後、復元されたタブを表示すると保存位置へ移動する
- [x] 保存位置が文書の末尾を超える場合はブラウザーの通常動作で現在の末尾へクランプされる
- [x] タブIDは起動ごとに変わっても、保存時のタブエントリに対応する位置が復元される

### US-002: Explorer展開状態の復元

As a ユーザー  
I want to Explorerで展開していたフォルダーが復元される  
So that フォルダー階層を開き直さずに済む

**受け入れ条件**

- [x] 現在のExplorerソースに属する展開中ディレクトリがセッションへ保存される
- [x] 復元時は親ディレクトリから順に子エントリを遅延読み込みして展開する
- [x] 削除・移動・読み込み失敗したディレクトリは黙ってスキップし、他の復元を継続する
- [x] 異常な保存データによる無制限な読み込みを避けるため、1回の復元件数に上限を設ける

### US-003: ファイル内検索状態の復元

As a ユーザー  
I want to ファイル内検索の条件が復元される  
So that 同じ検索を入力し直さずに済む

**受け入れ条件**

- [x] 検索バーの開閉、検索語、正規表現設定がセッションへ保存される
- [x] 復元後は現在の文書DOMを対象に検索を再実行する
- [x] 検索結果、マッチ件数、現在マッチ、エラーは保存せず再計算する

### US-004: 既存の復元確認フローとの統合

As a ユーザー  
I want to 追加のUI状態も既存の復元判断に従う  
So that セッションを破棄したのに一部だけ復元されることがない

**受け入れ条件**

- [x] リロード時はタブ等と一緒に追加UI状態も自動復元される
- [x] 完全再起動時は「復元する」を選んだ場合だけ追加UI状態が復元される
- [x] 「破棄する」を選ぶと追加UI状態を含む保存済みセッション全体が破棄される
- [x] 復元確認中の操作で保存済みセッションを上書きしない既存ガードを維持する
- [x] `Ctrl+R` / `Cmd+R` / `F5` は既定の再読込を止め、タブ・最近使った項目・設定の保存完了後に再読込する

## 非機能要求

- スクロール操作のたびにディスクI/Oを直接実行せず、既存の保存キューでまとめる
- 保存データは信頼せず、不正な型・非有限数・負数・過剰なExplorer展開数を復元しない
- `tabs.json` は現行スキーマだけを受理し、旧開発版の `activePath` / `rootPath` は読み書きしない

## 未決定事項（設計フェーズで決定）

- [x] スクロール位置をタブID以外のどの識別子へ紐づけるか
- [x] Explorerの遅延読み込みを伴う復元順序と件数上限
- [x] 検索状態のうち保存対象とする項目
