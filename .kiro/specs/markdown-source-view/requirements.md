# 要求定義: Markdownソース確認モード

## 背景・目的

レンダリング結果だけでは、リンク先、記法、frontmatter、図表定義など元のMarkdownを確認しづらい。編集機能は追加せず、現在のタブを読み取り専用のソース表示へ切り替えられるようにする。

## スコープ

- 通常レンダリングと元Markdownソースのタブ単位切替
- ステータスバーからの切替と共通コマンド
- ネイティブFSおよびZIP内Markdown
- ソース全文コピー、ページ内検索、安全な見出しアウトライン

## 対象外

- Markdownの編集・保存
- シンタックスハイライト、行番号、差分表示
- viewModeのセッション永続化
- 大容量セーフモードの置き換え

## US-001: 元Markdownを確認する

As a ユーザー
I want to レンダリング結果と元Markdownを切り替える
So that 記法や定義を外部エディターを開かずに確認できる

### 受け入れ条件

- [x] 通常表示中のアクティブタブがある場合、ステータスバーに現在の表示モードを示す
- [x] ステータスバー操作で「レンダー」と「ソース」を切り替えられる
- [x] 表示モードはタブごとに独立し、新規タブは「レンダー」で開始する
- [x] ソースは読み取り専用・等幅フォントで、Markdown文字列をHTMLとして解釈せず表示する
- [x] ソース表示中もファイル更新・ZIP再読み込み後の最新内容へ追従する
- [x] ネイティブFSとZIP内Markdownで同じように動作する
- [x] ソース全文を明示操作でクリップボードへコピーできる

## US-002: 既存ビューア機能との整合

As a ユーザー
I want to ソース確認中も閲覧操作を継続する
So that 表示切替のために現在の文脈を失わない

### 受け入れ条件

- [x] ソース表示中もページ内検索を利用できる
- [x] Rustが返した安全なアウトラインを使い、TOCから元Markdownの見出し位置へ移動できる
- [x] タブおよび表示モードごとに実行中のスクロール位置を保持する
- [x] ソース表示中は描画済みHTML・SVGに依存するコンテキストメニュー項目を表示しない
- [x] 印刷、選択コピー、外部エディター起動は既存方針どおり利用できる
- [x] Mermaid、KaTeX、画像hydration、外部画像読み込み等のレンダリング後処理をソースへ実行しない
- [x] 大容量セーフモード中は重複するソース切替を表示せず、既存の通常表示確認フローを維持する
- [x] ソース全文とコードブロックのコピー成功時に、同じ完了メッセージを一時表示する

## 非機能要求

- ソースはSvelteのテキストノードとして描画し、`{@html}`へ渡さない
- 新しいMarkdown読込経路やファイル権限を追加せず、`contentStore.raw`を表示する
- viewModeは既存の`renderMode`（full / safe）と別の型・状態にする
- ソース切替だけでは設定ファイルやタブ復元データを書き換えない

## 未決定事項（設計フェーズで決定）

- [x] 切替位置 → ステータスバー
- [x] 状態範囲 → タブ単位
- [x] 初期値 → レンダー
- [x] 永続化 → 今回は行わず、復元強化で再検討
- [x] ソース表示 → 軽量なプレーンテキスト、シンタックスハイライトなし
- [x] コピー → 全文コピーボタンを提供
