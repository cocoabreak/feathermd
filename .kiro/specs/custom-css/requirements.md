# 要求定義: カスタムCSS (custom-css)

## 背景・動機

ユーザーがMarkdown本文のフォント、色、余白などを調整できるようにする。一方でアプリUIの操作性とレンダラープラグインの拡張性を損なわない明確な適用境界を設ける。

## スコープ

- **対象**: Markdown本文、コードブロック、画像、KaTeX、Mermaid、将来のレンダラープラグイン、印刷/PDF
- **対象外**: タブ、サイドバー、目次、設定、検索、ライトボックス等のアプリUI
- **初期版で非対応**: `@import`、`url()`（ローカル画像・フォントを含む）

## ユーザーストーリー

### US-001: CSSファイルの選択と適用

- [x] 設定画面で単一のCSSファイルを選択・解除できる
- [x] カスタムCSSの有効/無効を切り替えられ、設定が永続化される
- [x] CSSはMarkdown本文とレンダラープラグインへ即時適用される
- [x] CSSセレクタは`.markdown-body`配下へ自動的にスコープされ、アプリUIに影響しない

### US-002: 自動再読み込みとエラー処理

- [x] 選択したCSSファイルの更新が再起動なしで反映される
- [x] CSSの構文エラー、`@import`、`url()`、読み込み失敗は設定画面に表示される
- [x] エラー時は直前の不正なCSSを適用せず、アプリの既定スタイルで操作を継続できる

### US-003: プラグイン互換性

- [x] KaTeXとMermaidがカスタムCSSの対象になる
- [x] fence型レンダラープラグインのルート要素に`viewer-plugin`クラスと`data-viewer-plugin`属性が付く
- [x] プラグイン作成ガイドにカスタムCSS境界の規約が記載される

## 非機能要求

- CSSの解析・スコープ変換は文字列置換ではなくCSSパーサーを使用する
- CSSファイルの読み込みと監視は既存の信頼済みルート境界に従う
- CSSファイルはUTF-8、拡張子`.css`、最大1MiBとする
