# 要求定義: ナビゲーション履歴（戻る/進む） (nav-history)

## 背景・動機

Wikiリンクやエクスプローラー・全文検索からファイルを次々開いていくと、「さっき見ていたファイルに戻りたい」場面が頻発する。
ブラウザやObsidianと同様の「戻る/進む」を提供し、閲覧の往復をスムーズにする。

2026-07-10に設計論点を確定して着手。

## スコープ

- **対象**: アクティブファイルの切替履歴の記録、「戻る」「進む」による履歴上の移動、Alt+←/→ ショートカット、マウスの戻る/進むボタン（XButton1/2）
- **対象外**:
  - 見出しアンカー・スクロール位置単位の履歴（ファイル単位のみ。必要になればv2で検討）
  - 履歴の永続化（アプリ再起動で履歴はクリアされる。セッション復元はタブのみ）
  - 履歴一覧UI（ボタン長押しでのドロップダウン等）。単純な戻る／進むボタンは`open-context-ui`の対象

## ユーザーストーリー

### US-001: 戻る/進む

```
As a ユーザー
I want to 直前に見ていたファイルへ「戻る」、戻った後に「進む」で復帰する
So that 複数ファイルの往復閲覧をタブ探しなしで行える
```

**受け入れ条件**

- [x] リンククリック・エクスプローラー選択・タブクリック・検索結果クリックなど、手段を問わず「表示ファイルが変わった」ことが1エントリとして記録される
- [x] Alt+← で履歴上の1つ前のファイルへ戻り、Alt+→ で1つ先のファイルへ進む
- [x] マウスの戻るボタン（XButton1）/進むボタン（XButton2）でも同じ操作ができる
- [x] 戻った状態から新しいファイルを開くと、それより先の「進む」履歴は破棄される（ブラウザと同じ挙動）
- [x] 同じファイルが連続して記録されることはない（重複エントリなし）
- [x] 履歴の先頭で「戻る」、末尾で「進む」を押しても何も起きない（エラーを出さない）

### US-002: 閉じたタブ・消えたファイルの扱い

```
As a ユーザー
I want to 戻る先のタブを閉じていても「戻る」で再び開ける
So that タブを閉じたことを気にせずナビゲーションできる
```

**受け入れ条件**

- [x] 戻る/進む先のファイルがタブとして開いていればそのタブをアクティブにする
- [x] タブが閉じられていればファイルを開き直す（新しいタブとして開く）
- [x] ファイル自体が削除されている等で開けない場合、そのエントリを履歴から除去し、同方向の次のエントリへ移動する（エラーダイアログを出さない）

## 未決定事項（設計フェーズで決定）

- [x] 履歴に記録する単位 → アクティブファイルの切替全部（手段を問わない）。閉じたタブは再オープン
- [x] 操作手段 → Alt+←/→ とマウスの戻る/進むボタン。画面上のボタンは`open-context-ui`で追加
- [x] アンカー単位の履歴 → v1はファイル単位のみ
