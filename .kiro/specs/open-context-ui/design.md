# 設計: オープンコンテキストUI

## 状態モデル

`tabStore` / `historyStore` は表示対象、`explorerStore.rootPath` は探索起点として独立させる。既存のセッション保存形式もこの分離を維持し、新たな「open source」フィールドは追加しない。

## UI

- サイドバー上部に戻る、進む、ファイルを開く、フォルダーを開く操作を置く。
- Explorer表示中、アクティブファイルがルート外（またはルートなし）なら、その親フォルダーを開く案内を表示する。
- ビューアーの空状態にファイル／フォルダーの主要ボタン、D&D案内、最近開いたファイルを表示する。

## パス判定とセキュリティ

ルート内外の表示判定には`isWithinDir`を使うが、これはUX判定に限定する。「親フォルダーを開く」は既存の`openFolder`を経由し、Rust側のcanonicalize、AllowedRoots、危険ルート拒否を最終境界とする。

## 履歴

戻る／進むボタンは既存のコマンドを実行する。`historyStore.canGoBack` / `canGoForward`を無効状態に使い、ナビゲーションでExplorerルートは変更しない。
