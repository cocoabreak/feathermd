# 要求定義: オープンコンテキストUI (open-context-ui)

## 背景

ファイル単体、フォルダー、Explorer、履歴など複数の経路からファイルを開けるため、表示中ファイルとExplorerルートの関係を予測可能にする。あわせて、ネイティブメニューを知らなくても主要操作へ到達できるUIを提供する。

## 基本方針

- Explorerルートを変更するのは、ユーザーが明示的に「フォルダーを開く」を実行した場合だけとする。
- ファイルを開く、タブを切り替える、戻る／進むを実行する操作ではExplorerルートを変更しない。
- 「Explorerから開いたか」「ファイル選択から開いたか」という出所は永続化しない。閲覧履歴は表示ファイルのパス、Explorer履歴は明示的に開いたルートのパスを保持する。
- 親子関係にあるフォルダーも別々の履歴項目として保持し、自動統合しない。

## 受け入れ条件

- [x] 画面上から「ファイルを開く」「フォルダーを開く」を実行できる。
- [x] ファイル未表示時にドラッグ＆ドロップでも開けることが分かる。
- [x] 画面上の戻る／進むボタンでファイル履歴を移動でき、移動不能時は無効になる。
- [x] 表示中ファイルがExplorerルート外、またはルート未設定の場合、「親フォルダーを開く」を明示的に実行できる。
- [x] 「親フォルダーを開く」を押すまでExplorerルートは変化しない。
- [x] 親フォルダーを開く際も既存の信頼確認・危険フォルダー拒否を通る。
- [x] セッション復元時もタブとExplorerルートは独立して復元される。

## 対象外

- 開いた経路の永続化・表示
- Explorerツリー内の表示中ファイルへの自動展開・自動スクロール
- 履歴一覧のドロップダウンUI
