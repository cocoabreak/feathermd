# 要求定義: タブ操作の改善

## 背景・目的

複数のMarkdownを比較する際に、誤って閉じたタブの復元、不要なタブの一括整理、目的の順序への並べ替えを可能にする。既存のピン留めとセッション復元を拡張し、操作経路をタブのコンテキストメニュー、ショートカット、コマンドパレットへ統一する。

## スコープ

- 閉じたタブを最大10件、アプリ実行中のみ履歴として保持して再オープン
- 対象タブ以外、または対象タブより右側にある未ピン留めタブの一括クローズ
- ドラッグによるタブの並べ替え
- ピン留めタブの先頭グループ化
- 既存セッション保存による並び順・ピン留め状態の復元

## 対象外

- 閉じたタブ履歴の再起動後の永続化
- タブの複数選択、別ウィンドウへの移動、分割表示
- ピン留めタブを一括クローズする操作

## US-001: 閉じたタブを復元する

- [x] `Ctrl+Shift+T`、アプリメニュー、コマンドパレットから直近のタブを再オープンできる
- [x] クローズボタン、`Ctrl+W`、一括クローズで閉じたタブを最大10件まで新しい順に保持する
- [x] 再オープン時は既存のDocumentSource、AllowedRoots、サイズ確認、ZIP再検証を通して内容を読み直す
- [x] 元の表示モードと可能な範囲のタブ位置を復元する
- [x] ファイルが削除・アクセス不能の場合はタブを作らず、該当履歴を消費する

## US-002: タブをまとめて閉じる

- [x] タブの右クリックメニューとコマンドパレットから「他のタブを閉じる」を実行できる
- [x] タブの右クリックメニューとコマンドパレットから「右側のタブを閉じる」を実行できる
- [x] ピン留めタブは一括クローズの対象にしない
- [x] 閉じた各タブでファイル監視・ZIPソース・コンテンツを通常のクローズと同様に解放する

## US-003: タブを並べ替えて固定する

- [x] タブをドラッグして同じグループ内の任意の位置へ移動できる
- [x] ピン留めすると先頭のピン留めグループ末尾へ移動し、解除すると通常グループ先頭へ移動する
- [x] ピン留めタブと通常タブの境界を越えるドラッグは行わない
- [x] タブ順序とピン留め状態は既存セッション保存で再起動・リロード後に復元される
- [x] 並べ替え後もアクティブタブ、`Ctrl+Tab`、`Ctrl+数字`は表示順に従う

## 非機能要求

- 新しい依存関係、Tauri capability、永続化ファイルを追加しない
- 一括クローズは既存の安全なクローズ経路を再利用する
- ドラッグ中もタブのボタン操作と横スクロールを損なわず、既存のTauriネイティブファイルドロップを維持する
