# 要求定義: 大容量文書のセーフモード

## 背景

実WebView計測では、通常レンダリングが1MiBで約4.4秒、5MiBで約86秒かかり、10MiBでは180秒以内に完了しなかった。既存の`large-markdown-warning` specは明示確認なしの通常レンダリングを防ぐが、承認後の操作不能は解消しない。

大容量文書でも内容を確認できるよう、リッチなMarkdown変換を行わず、ソースを安全なプレーンテキストとして表示するセーフモードを追加する。

## US-001: 大容量文書をセーフモードで開く

- [x] 5MiB以上10MiB以下のMarkdownを開くと、通常レンダリングの代わりにセーフモードで開くか確認する
- [x] 確認にはファイル名、セーフモードが推奨であること、リッチな表示が無効になることを表示する
- [x] ユーザーが承認した場合、タブをセーフモードで開く
- [x] ユーザーが拒否した場合、タブ・最近使ったファイル・watcher・本文キャッシュを変更しない
- [x] 10MiBを超えるファイルは従来どおり拒否する

## US-002: セーフモードを低負荷かつ安全に表示する

- [x] MarkdownソースをHTMLとして解釈せず、エスケープされたプレーンテキストとして表示する
- [x] Shiki、KaTeX、Mermaid、HTMLレンダリング、画像読込、プラグインpostRender、コードコピー、行番号、ライトボックスを実行しない
- [x] 改行と空白を維持し、長い行はコンテンツ領域内で折り返す
- [x] セーフモードであることと、無効になっている主な機能を本文上部に表示する
- [x] TOC、frontmatter、読書統計、検索ハイライト等、直前の通常表示から派生した状態を残さない
- [x] ページ内検索と生成済みHTMLを前提とするexport操作はセーフモード中無効にし、その理由を利用者へ示す
- [x] セーフモード表示は既存のHTMLサニタイズやローカルファイル認可を迂回しない

## US-003: 明示操作で通常表示へ切り替える

- [x] セーフモードの案内から通常表示を要求できる
- [x] 通常表示の前に、長時間の応答停止が起こり得ることを改めて確認する
- [x] 確認を承認した場合だけ、同じタブを通常表示へ切り替える
- [x] 通常表示を承認したパスはアプリ実行中だけ記憶し、同じファイルの監視更新では再確認しない
- [x] 通常表示の確認を拒否した場合、セーフモードを維持する

## US-004: タブと更新で表示モードを維持する

- [x] セーフモードか通常表示かをタブ単位で保持する
- [x] タブ切り替え、戻る／進む、ファイル監視による再読込で現在の表示モードを維持する
- [x] 通常表示中の小容量文書が監視更新で5MiB以上になった場合、確認ダイアログを連発せず自動的にセーフモードへ切り替える
- [x] セッションへ通常表示の承認状態を永続化しない
- [x] アプリ再起動後に大容量文書を復元する場合は、再度セーフモードの確認を行う
- [x] 小容量文書の既存の表示・復元・監視動作を変更しない

## US-005: セーフモードでも目次を利用する

- [x] 5MiB以上の文書はRust側のCommonMark/GFMパーサーで見出しを抽出する
- [x] 見出しのレベルとインライン装飾を除いた表示文字列を目次へ表示する
- [x] 目次を選択するとプレーンテキスト内の見出し位置へ移動する
- [x] 見出しアンカー以外のMarkdown HTMLを生成しない
- [x] 見出しは先頭2,000件を上限とし、上限到達時は目次が省略されていることを表示する

## 非機能要求

- セーフモードの本文DOMは入力行数や段落数に比例させず、見出しアンカーも最大2,000件に制限する
- 5MiBおよび10MiBの実測用文書で、通常表示より大幅に短い時間で内容を確認でき、表示後にアプリ操作を継続できることを確認する
- セーフモード中は本文に含まれるHTML、画像URL、リンクURLを実行・取得しない
- 確認ダイアログは同一パスの並行要求を重複表示せず、エラー時はキャンセルとしてfail-closedに扱う
- 日本語・英語の表示に対応する

## スコープ外

- Markdownの仮想化レンダリング、ストリーミング構文解析
- 10MiBを超えるMarkdownの読込
- 通常レンダリング自体の高速化
- ユーザーが小容量文書を任意にセーフモードへ切り替える機能
