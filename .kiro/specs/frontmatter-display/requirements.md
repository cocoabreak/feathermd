# 要求定義: frontmatter（YAML）のメタデータ表示 (frontmatter-display)

## 背景・動機

Markdownファイル先頭のYAML frontmatter（`---`で囲まれたメタデータブロック）が、現状は素通しでMarkdown本文として扱われ、水平線や崩れた文字列として表示されてしまう。メタデータとして解釈し、分かりやすい形で表示したい。

## スコープ

- **対象**: ファイル先頭のYAML frontmatterブロックの検出・解析・本文からの除外・メタデータ表示
- **対象外**: frontmatterの編集（本アプリはビューワー専用）、TOML/JSON形式のfrontmatter、ネストの深い構造の凝った表示（フォールバックでの簡易表示に留める）

---

## ユーザーストーリー

### US-001: frontmatterのメタデータ表示

```
As a ユーザー
I want to ファイル先頭のYAML frontmatterをメタデータとして分かりやすく確認する
So that 本文と区別しつつ、そのファイルの属性（タイトル・日付・タグ等）を把握できる
```

**受け入れ条件**

- [x] frontmatterブロックはMarkdown本文としてレンダリングされず、崩れた表示（水平線化等）が起きない
- [x] frontmatterのキーと値が一覧（ラベル: 値）として表示される
- [x] 値が文字列/数値の配列（タグ等）の場合、pill（バッジ）形式で表示される
- [x] frontmatterが存在しないファイルでは、メタデータ表示自体が現れない

### US-002: 不正なYAMLでもクラッシュしない

```
As a ユーザー
I want to frontmatterの記法が誤っていてもアプリが壊れない
So that 安定して他のファイルも閲覧できる
```

**受け入れ条件**

- [x] YAMLとして解析できないfrontmatterブロックは、メタデータ表示を出さずに警告ログのみ出力する
