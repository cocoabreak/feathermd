# 要求定義: ファイル内検索 (in-file-search)

## 背景・動機

現状、閲覧中のMarkdown文書内で特定の語句を探す手段がない（ブラウザ標準のCtrl+Fに相当する機能が未実装）。長い文書やAPIリファレンスのような文書では、TOC（見出し単位）だけでは本文中の特定語句を見つけにくい。

## スコープ

- **対象**: 現在アクティブなタブ（レンダリング済みプレビュー）内でのテキスト検索、マッチ箇所のハイライト表示、次候補/前候補へのジャンプ、正規表現検索
- **対象外**: 複数タブ横断検索、Markdownソース（生テキスト）に対する検索、検索結果の一括置換（本アプリは編集機能を持たないビューワーのため対象外）、ファイル名検索（エクスプローラーのフィルタ機能とは別領域）

---

## ユーザーストーリー

### US-001: キーワード検索とハイライト

```
As a ユーザー
I want to 表示中のMarkdown文書内でキーワードを検索する
So that 長い文書の中から目的の箇所をすぐに見つけられる
```

**受け入れ条件**

- [x] `Ctrl+F` で検索バーを開ける
- [x] 検索語を入力すると、一致箇所がすべてハイライト表示される
- [x] マッチ件数（例: `3/12`）が表示される
- [x] 一致なしの場合はその旨がわかる表示になる
- [x] `Escape` または閉じるボタンで検索バーを閉じ、ハイライトも消える

---

### US-002: マッチ間のジャンプ

```
As a ユーザー
I want to 一致箇所を1件ずつ前後に移動する
So that 多数のマッチがあっても目的の箇所に素早くたどり着ける
```

**受け入れ条件**

- [x] 次候補ボタン（または `Enter`）で次のマッチへスクロール・現在地ハイライトが移動する
- [x] 前候補ボタン（または `Shift+Enter`）で前のマッチへ移動する
- [x] 最後のマッチの次は先頭のマッチに、先頭の前は最後のマッチにループする
- [x] 現在地のマッチは他のマッチと異なる強調表示になる

---

### US-003: 正規表現検索

```
As a ユーザー
I want to 正規表現で検索する
So that パターンマッチによる高度な検索ができる
```

**受け入れ条件**

- [x] 正規表現モードのトグルがある
- [x] 正規表現モード時、入力文字列は正規表現として評価される
- [x] 不正な正規表現を入力した場合はエラー表示になり、アプリがクラッシュしない

---

### US-004: タブ切り替え時の検索対象追従

```
As a ユーザー
I want to 検索バーを開いたままタブを切り替えても、常に今表示中のタブが検索対象になる
So that タブごとに検索を開き直す手間がない
```

**受け入れ条件**

- [x] 検索バーを開いたまま別タブに切り替えると、新しいアクティブタブの内容に対して同じ検索語で再検索される
- [x] 対象タブに一致箇所がなければマッチ0件の表示になる

---

## 非機能要求

| 項目 | 要求 |
|------|------|
| 応答性 | 検索語入力からハイライト反映までの体感遅延が小さいこと（1MB程度のファイルでも極端な入力遅延が出ない） |
| 安全性 | 不正な正規表現・過度に高コストな正規表現を入力してもアプリ全体がフリーズ/クラッシュしない（自分自身の入力によるものなので、ネットワーク越しの悪意ある入力からの防御は対象外） |

---

## 未決定事項（設計フェーズで決定）

- [x] 大文字小文字の区別の扱い（デフォルト大文字小文字無視とする案）
- [x] 検索バーのUI配置（コンテンツ領域上部オーバーレイを想定）
