# 要求定義: リリース確認 (version-check)

## 背景・目的

FeatherMDの新しい安定版が公開されたことを利用者へ知らせる。ただし、更新ファイルの
ダウンロード、インストール、再起動は行わず、利用者がGitHub Releasesを開いて判断する。

## スコープ

- 起動時のバックグラウンド確認（既定で有効、設定で無効化可能）
- Aboutダイアログからの手動確認
- 新版がある場合の非モーダル通知
- GitHub Releasesページを既存openerで開く導線

対象外:

- Tauri updaterの導入
- 更新ファイルの自動ダウンロード・検証・インストール
- draft、prerelease、他リポジトリ、任意URLの確認

## 受け入れ条件

- [x] 起動完了後、設定が有効な場合だけ最新安定版を1回確認する
- [x] 自動確認は起動処理をブロックせず、最新版・通信失敗時には通知しない
- [x] 新版がある場合だけ非モーダル通知を表示し、GitHub Releasesを開ける
- [x] 設定画面で起動時確認を無効化でき、既定値は有効とする
- [x] Aboutからは設定にかかわらず手動確認・再確認できる
- [x] Aboutでは確認中、最新版、新版あり、失敗を表示する
- [x] `vX.Y.Z`と`X.Y.Z`をSemVerとして比較する
- [x] 通信先、タイムアウト、レスポンス量、遷移先URLを固定・制限する
- [x] 自動更新処理を実装しない
