# 要求定義: ネイティブUI化 (native-ui)

## 背景・動機
現在のUIはブラウザのデフォルトのウィンドウ枠（タイトルバーなど）を使用している可能性がある。
アプリにより深い没入感とデスクトップアプリらしい洗練された外観（プレミアム感）を持たせるため、OSの標準ウィンドウ枠を非表示（またはカスタマイズ）し、Tauriの機能を用いたカスタムタイトルバーやネイティブライクなUIを提供する。

## ユーザーストーリー
### US-001: カスタムタイトルバーの表示
```
As a ユーザー
I want to アプリ独自のデザインに統合されたタイトルバーを使用する
So that アプリ全体が統一された美しいデザインで利用できる
```
- [x] OS標準のタイトルバーが非表示になっている
- [x] アプリのトップにドラッグ可能なカスタムタイトルバーが存在する
- [x] タイトルバーに最小化、最大化、閉じるのカスタムボタンが存在する
- [x] タイトルバー左端のメニューボタンから既存のネイティブアプリメニューを表示できる

### US-002: ネイティブなウィンドウコントロール
```
As a ユーザー
I want to OSの標準的なウィンドウ操作（スナップ、リサイズ等）を損なわずに行える
So that 操作感を犠牲にせずに美しいUIを享受できる
```
- [x] カスタムタイトルバーであっても、Windowsのスナップ機能（画面端へのドラッグ等）が動作する
- [x] ウィンドウの境界線でリサイズが可能である
