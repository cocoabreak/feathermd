# 要求定義: キーボードショートカット (keyboard-shortcuts)

## 背景・動機

現状、ファイルを開く・タブ切り替え・パネル表示切替はすべてマウス操作（Toolbarのボタンクリック等）でのみ行える。
VS Codeなど既存エディタに近いショートカットを提供し、キーボード中心の操作を可能にすることで操作効率を上げる。

## スコープ

- **対象**: ファイル/フォルダを開く、タブ操作、パネル表示切替、設定パネルの開閉に対するショートカット
- **対象外**: キーバインドをユーザーが変更できるUI（将来検討）、macOS向けのCmdキー対応（現状Windows優先のためv1スコープ外）

**アーキテクチャ上の前提**
将来のキーバインドカスタマイズを見据え、「コマンド（操作の実体）」と「キーバインド（キーcombo→コマンドidの対応表）」を分離した設計とする。
v1ではキーバインドはコード内の固定テーブルとして持つが、コマンド実行の経路を一本化しておくことで、将来キーバインドを設定ファイル化・UI編集可能にする際にコマンド側の実装変更が不要になるようにする。
Toolbarボタン等の既存のマウス操作も、可能な範囲で同じコマンドを呼び出す形に統一する。

## ユーザーストーリー

### US-001: ファイル/フォルダを開くショートカット

```
As a ユーザー
I want to キーボードショートカットでファイル/フォルダを開くダイアログを呼び出す
So that マウスでToolbarボタンを探さずに操作できる
```

**受け入れ条件**

- [x] `Ctrl+O` でファイルを開くダイアログが開く
- [x] `Ctrl+Shift+O` でフォルダを開くダイアログが開く
- [x] 入力フォーカスがテキスト入力欄（将来的な検索ボックス等）にある場合はショートカットを発火させない

---

### US-002: タブ操作ショートカット

```
As a ユーザー
I want to キーボードでタブの切り替え・クローズを行う
So that 複数ファイルを開いている際にマウス移動なしで移動できる
```

**受け入れ条件**

- [x] `Ctrl+Tab` で次のタブへ、`Ctrl+Shift+Tab` で前のタブへ切り替わる（末尾/先頭でループする）
- [x] `Ctrl+W` でアクティブタブを閉じる
- [x] `Ctrl+1`〜`Ctrl+9` でN番目のタブに直接切り替わる（該当タブがない場合は何もしない）

---

### US-003: パネル表示切替ショートカット

```
As a ユーザー
I want to キーボードでSidebar・TOCPanelの表示/非表示を切り替える
So that 閲覧に集中したいときにすぐパネルを隠せる
```

**受け入れ条件**

- [x] `Ctrl+B` でSidebarの表示/非表示が切り替わる
- [x] `Ctrl+J` でTOCPanelの表示/非表示が切り替わる

---

### US-004: 設定パネルの開閉ショートカット

```
As a ユーザー
I want to キーボードで設定パネルを開閉する
So that マウス操作なしで設定を確認・変更できる
```

**受け入れ条件**

- [x] `Ctrl+,` で設定パネルが開く
- [x] 設定パネル表示中に `Esc` で閉じる

---

## 非機能要求

| 項目 | 要求 |
|------|------|
| 競合回避 | ブラウザ/OS標準ショートカットと衝突する場合はWebView内でのデフォルト動作を明示的に抑止する（例: `Ctrl+O` のファイルダイアログ標準動作） |
| スコープ | ショートカットはアプリウィンドウにフォーカスがある場合のみ有効 |
