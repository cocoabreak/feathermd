# 要求定義: UI細部改善（フォルダを閉じる・履歴削除・目次リサイズ） (ui-refinements)

## 背景・動機

日常利用で気になった細かな操作性の不足を3点まとめて改善する（2026-07-11 ユーザー要望）。

1. ファイルは全タブを閉じると「最近開いたファイル」から選び直せるが、フォルダには「閉じる」操作自体がなく、同じ流れで「最近開いたフォルダー」に戻れない
2. 最近開いたファイル・フォルダーの一覧から不要な履歴を消せない（recent-files specでは対象外としていた）
3. 目次の高さ調整が目次パネル上端の細いハンドルでしか行えず、狙いにくい

## スコープ

- **対象**:
  - ネイティブメニューからエクスプローラーのルートフォルダを閉じる操作
  - 最近開いたファイル・フォルダー一覧の項目の個別削除
  - 目次ヘッダーバー（「目次」の見出し行）のドラッグによる高さ調整
- **対象外**:
  - 履歴の一括クリア操作
  - 存在しなくなったパスの自動除去
  - フォルダを閉じた際のタブへの影響（タブは開いたまま維持する）

---

## ユーザーストーリー

### US-001: フォルダを閉じる

```
As a ユーザー
I want to 開いているフォルダをメニューから閉じる
So that ファイルと同様に「最近開いたフォルダー」一覧から次のフォルダへ移れる
```

**受け入れ条件**

- [x] ファイルメニューに「フォルダを閉じる」項目があり、実行するとエクスプローラーのツリーが空になる
- [x] フォルダを閉じた後、最近開いたフォルダーがあればサイドバーに一覧が表示される
- [x] フォルダを閉じるとディレクトリ監視が解除される
- [x] フォルダを閉じた状態がセッションに保存される（再起動・リロードで閉じたままになる）
- [x] 開いているタブはフォルダを閉じても影響を受けない
- [x] フォルダを開いていない状態で実行しても何も起きない（エラーにならない）

---

### US-002: 最近開いた履歴の個別削除

```
As a ユーザー
I want to 最近開いたファイル・フォルダー一覧から不要な項目を消す
So that 使わなくなったパスや誤って開いたパスが一覧に残り続けない
```

**受け入れ条件**

- [x] 一覧の項目にマウスオーバーすると、左端に削除ボタン（×）が表示される
- [x] 削除ボタンをクリックすると該当項目だけが一覧から消え、その項目は開かれない
- [x] 削除結果は永続化され、再起動後も削除されたままになる
- [x] 「最近開いたファイル」（ビューワー空状態）・「最近開いたフォルダー」（サイドバー空状態）の両方で削除できる

---

### US-003: 目次ヘッダーバーでの高さ調整

```
As a ユーザー
I want to 目次の見出しバーそのものをドラッグして高さを調整する
So that 細いハンドルに正確にカーソルを合わせなくて済む
```

**受け入れ条件**

- [x] 目次ヘッダーバー（「目次」行）を上下にドラッグすると、既存の上端ハンドルと同様に高さ（比率）が変わり永続化される
- [x] ヘッダーバーのダブルクリックでデフォルト比率に戻る（既存ハンドルと同挙動）
- [x] ヘッダーバー右端の✕（目次を閉じる）ボタンは従来どおりクリックで動作し、ドラッグ開始の対象にならない
- [x] 目次が空（ヘッダー行のみ表示）のときはドラッグ・ダブルクリックとも無効
- [x] 既存の上端ハンドルの挙動は変わらない

---

## 非機能要求

なし

## 未決定事項（設計フェーズで決定）

- [x] サイドバー空状態の判定基準（`tree.length === 0` か `rootPath === null` か）
