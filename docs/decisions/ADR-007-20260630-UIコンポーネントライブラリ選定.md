# ADR-007: UIコンポーネントライブラリ選定（shadcn-svelte）

- **Date**: 2026-06-30
- **Status**: Accepted

## コンテキスト

Svelte + Tailwind CSS の構成でUIコンポーネントライブラリを選定する。
候補: shadcn-svelte / 素のTailwind CSS（ゼロから実装）

## 決定事項

**shadcn-svelte を採用する。**

## 理由

- Tailwind CSS前提の設計であり、ADR-006の技術選定と完全に一致する
- ダイアログ・ドロップダウン・スクロールエリア・トグルなど、本アプリで必要なUI要素がアクセシビリティ対応済みで揃っている
- 「コピペ型」コンポーネント（依存ライブラリとしてではなくソースをプロジェクトに取り込む形式）のため、自由にカスタマイズできる
- ロックインリスクが低い（外部パッケージへの依存がなく、コードはプロジェクト内に存在する）

## 影響・トレードオフ

- **メリット**: アクセシビリティ対応・Tailwindとの親和性・カスタマイズ自由度
- **デメリット**: コンポーネントをプロジェクト内に取り込むため、shadcn-svelteのアップデートを自動では受け取れない（手動で差分を取り込む必要がある）
- Radix UIベースのアクセシビリティ実装を活用することで、キーボード操作・スクリーンリーダー対応が担保される
