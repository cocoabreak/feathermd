# ADR-006: UIスタイリング・外観・コードハイライト選定

- **Date**: 2026-06-30
- **Status**: Accepted

## コンテキスト

フロントエンド（Svelte）のスタイリング方針、ウィンドウ外観、コードシンタックスハイライトのライブラリを選定する。

## 決定事項

| 項目 | 決定 |
|---|---|
| スタイリング | **Tailwind CSS** |
| ウィンドウ外観 | **OSネイティブタイトルバー** |
| コードハイライト | **shiki** |

## 理由

### Tailwind CSS
- ユーティリティクラスによりSvelteコンポーネント内でスタイルを完結させられる
- shadcn-svelte（Tailwind前提のUIコンポーネント集）の利用が可能になる
- デザイントークン（カラー・スペーシング）の一元管理がしやすい

### OSネイティブタイトルバー
- 実装コストが低く、各OSの操作感（ウィンドウ移動・最大化・閉じる）に自然に馴染む
- カスタムフレームレスはドラッグ領域の実装など追加コストが大きい
- ビューワーとしての本質的な価値に集中するためシンプルな選択を優先

### shiki
- VS Codeと同じ文法定義（TextMate grammar）を使用するため表示品質が高い
- テーマをVS Codeのものと統一できるため、ユーザーが馴染みやすい
- バンドルサイズはhighlight.jsより大きいが、デスクトップアプリとして許容範囲

## 影響・トレードオフ

- Tailwind CSS: ユーティリティクラスが多くなるが、Svelteのscoped styleと併用することで管理可能
- shiki: 初期ロード時にWasmモジュールのロードが発生するが、一度読み込めばキャッシュされる
- OSネイティブ: カスタムUIデザインの自由度がタイトルバー部分に限って制限される
