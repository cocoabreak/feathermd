# ADR-005: TeXレンダラー選定（KaTeX）

- **Date**: 2026-06-30
- **Status**: Accepted

## コンテキスト

Markdown内の数式（TeX記法）をレンダリングするライブラリを選定する。
候補: KaTeX / MathJax

## 決定事項

**KaTeX を採用する。**

## 理由

- KaTeXはMathJaxに比べてレンダリング速度が大幅に速い（同期的・ブロッキングなし）
- デスクトップのMarkdownビューワーとして、ファイル切り替え時の再描画速度は重要
- 一般的なMarkdownで使用されるTeX記法はKaTeXでほぼカバーできる
- SSRが不要なため、MathJaxのレンダリング後の再フロー問題が発生しない

## 影響・トレードオフ

- **メリット**: 高速・軽量・シンプルなAPI
- **デメリット**: MathJaxに比べてサポートするTeX命令が少ない。高度な数式（一部のAMS命令等）で非対応のケースがある
- 非対応命令に遭遇した場合はエラー表示でフォールバックし、他のコンテンツ表示には影響させない（US-006の受け入れ条件に準拠）
