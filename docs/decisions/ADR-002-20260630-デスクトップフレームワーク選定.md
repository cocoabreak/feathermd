# ADR-002: デスクトップフレームワーク選定（Tauri vs Wails）

- **Date**: 2026-06-30
- **Status**: Accepted

---

## コンテキスト

Markdownビューワーはデスクトップアプリとして実装する（US-001〜007）。
候補として挙がっているのは以下の2フレームワーク：

- **Tauri** — Rustバックエンド + システムWebView
- **Wails** — Goバックエンド + システムWebView

いずれもネイティブWebViewを活用し、フロントエンドはWeb技術で実装する点は共通。
バックエンド言語と周辺エコシステムが主な差異となる。

### 本プロジェクトで必要な機能

| 必要機能 | 用途 |
|---|---|
| ファイルシステムアクセス | ファイルオープン、ディレクトリ読み取り |
| ファイル監視（FSWatch） | US-004: ファイル更新の自動検知 |
| 外部プロセス起動 | US-006: Krokiローカルなど外部プロセス型レンダラー |
| OSブラウザ起動 | US-005: httpsリンクをブラウザで開く |
| クロスプラットフォームビルド | Windows（Must）/ Linux（Nice）/ macOS（Low） |

---

## 選択肢の比較

### Tauri（Rust）

| 観点 | 評価 |
|---|---|
| バックエンド言語 | Rust（型安全・高パフォーマンスだが習得コストが高い） |
| バンドルサイズ | 非常に小さい（数MB〜10MB程度） |
| Windows WebView | WebView2（Chromium基盤、Win11標準搭載） |
| Linux WebView | WebKitGTK（webkit2gtk） |
| ファイル監視 | `notify` クレートが成熟（クロスプラットフォーム対応） |
| 外部プロセス起動 | `std::process::Command` / `tokio::process` で対応可能 |
| セキュリティ | 権限モデルが明確（CSP・IPC許可リスト） |
| コミュニティ | 大きい（GitHub ★90k超） |
| 成熟度 | v2が安定稼働中 |
| 懸念点 | Rustの学習コスト。コンパイル時間が長い |

### Wails（Go）

| 観点 | 評価 |
|---|---|
| バックエンド言語 | Go（習得しやすく、並行処理が得意） |
| バンドルサイズ | 小さい（20〜30MB程度） |
| Windows WebView | WebView2（同上） |
| Linux WebView | WebKitGTK（同上） |
| ファイル監視 | `fsnotify` で対応可能（クロスプラットフォーム） |
| 外部プロセス起動 | `exec.Command` + goroutineで対応可能 |
| セキュリティ | Tauriより緩やか（明示的な権限モデルなし） |
| コミュニティ | 中規模（GitHub ★25k程度） |
| 成熟度 | v2安定、v3開発中 |
| 懸念点 | Tauriに比べてエコシステムが小さい。v3への移行期 |

---

## 共通事項

- **Windowsのレンダリング**: 両者ともWebView2（Chromium）を使用するため、Windows上での表示は一致する
- **Linuxのレンダリング**: 両者ともWebKitGTK。WindowsのWebView2と細部で差異が出る可能性がある
- **フロントエンド**: React / Vue / Svelteなど任意のWebフレームワークが使用可能（どちらを選んでも変わらない）
- **プラグイン型レンダラー（外部プロセス）**: 両者とも実現可能。設計難易度に差はない

---

## 決定事項

**Tauri（Rustバックエンド）を採用する。**

### 決定理由

- RustとGoどちらにも習熟しており、言語的な制約はない
- 開発スピードよりも品質・長期的な資産性を重視する方針
- Rustの型安全性とTauriのセキュリティモデルはファイルビューワーとして適切
- コミュニティ・サンプル・プラグインの豊富さが長期メンテナンスを支える
- Windows優先という要件においてWebView2（Chromium）との実績が豊富

---

## 影響・トレードオフ

### Tauriを選んだ場合
- **メリット**: 小バンドル・高セキュリティ・大コミュニティ
- **デメリット**: Rustの習得コスト、コンパイル時間（開発体験への影響）

### Wailsを選んだ場合
- **メリット**: Goの書きやすさ、goroutineによる並行処理の簡潔さ
- **デメリット**: Tauriより小さいエコシステム、v3移行期のリスク
