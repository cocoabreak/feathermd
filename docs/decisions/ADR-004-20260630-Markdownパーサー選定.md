# ADR-004: Markdownパーサー選定（markdown-it）

- **Date**: 2026-06-30
- **Status**: Accepted

## コンテキスト

Markdownのパースとレンダリングに使用するライブラリを選定する。
候補: marked / markdown-it / remark (unified)

プラグイン型レンダラーシステム（US-006）を実現するため、コードブロックのレンダリングを差し替えられる拡張ポイントが必要。

## 決定事項

**markdown-it を採用する。**

## 理由

- CommonMark準拠かつGFM（テーブル・タスクリスト等）をプラグインで対応できる
- `markdown-it-mermaid` / `@traptitech/markdown-it-katex` など、v1で必要なJS内蔵型レンダラーとの連携プラグインが既に存在する
- プラグインAPIがシンプルで、カスタムレンダラー（コードブロックの差し替え）の実装コストが低い
- remarkほど複雑なセットアップが不要で、v1の実装に適切

## 影響・トレードオフ

- **メリット**: 豊富なプラグイン・シンプルなAPI・高速
- **デメリット**: remarkに比べるとAST操作の柔軟性が低い。将来、高度なAST操作が必要になった場合はremarkへの移行を検討する
- **設計上の制約**: レンダラープラグインのインターフェースをパーサー非依存に設計し、将来の移行コストを最小化する（`design.md` §4参照）
