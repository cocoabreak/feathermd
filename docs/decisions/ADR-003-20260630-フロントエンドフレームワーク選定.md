# ADR-003: フロントエンドフレームワーク選定（Svelte）

- **Date**: 2026-06-30
- **Status**: Accepted

## コンテキスト

Tauri（Rust）採用により、フロントエンドはWebView上で動作するWebアプリとして実装する。
候補: React / Vue 3 / Svelte

## 決定事項

**SvelteKit（`ssr=false` + `adapter-static`によるSPA構成）を採用する。**

当初は「SvelteKitなし、Vite + SvelteのみのSPA構成」を想定していたが、
`create-tauri-app` の公式テンプレートがSvelteKit + adapter-staticを採用しており、
これがTauriの公式推奨構成であるため、これに従う。

## 実装構成

- `+layout.ts` に `export const ssr = false` を設定（SSR無効）
- `svelte.config.js` に `adapter-static` + `fallback: "index.html"` を設定（SPA出力）
- 結果として「Svelte + Vite SPA」と機能的に等価

## 理由

- TauriアプリはNode.jsサーバーを持たないため `ssr = false` にすることでSSRを無効化できる
- `adapter-static` が静的HTML/JS/CSSを出力し、Tauriがそのファイルを直接提供する
- この構成はTauri公式ドキュメントで明示的に推奨されている
- SvelteKitは内部でViteを使用しており、当初想定と実質同等

## 影響・トレードオフ

- **メリット**: 最小バンドルサイズ・高いリアクティビティ・Tauriとの親和性・公式サポート
- **デメリット**: ReactやVueに比べてエコシステムが小さい
- SvelteKitの`routes/`構造が存在するが、アプリは単一ページのため実質的な影響はない
- UIコンポーネントライブラリ: shadcn-svelte（ADR-007）
